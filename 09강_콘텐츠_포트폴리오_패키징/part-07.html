💬 그대로 읽으세요
지금부터 팀별로 3분씩 발표합니다. 완성도를 평가하는 자리가 아니에요. "어떤 사례를 골랐고, 어떤 포맷으로 정리했는지" 를 공유하는 시간입니다. 다른 팀 아이디어를 보는 것 자체가 학습이에요.

각 팀의 편집장 또는 웹 담당이 대표해서 다음 4가지만 공유해 주세요.

1. 대표 사례 제목 (30초) 2. Before → After 한 줄씩 (60초) 3. PDF · 웹 시안 화면 공유 (60초) 4. 남는 숙제 한 줄 (30초 · "○○ 부분이 아직 완성이 안 됐습니다")

💬 그대로 읽으세요
Phase 2를 지나면서 "콘텐츠를 만드는 감각" 이 손에 붙으셨을 겁니다. 이제 다음 단계로 갑니다.

Phase 2까지는 콘텐츠를 하나하나 직접 만드는 구간이었습니다. 손으로 만든 결과물이 네 개 쌓였죠. 문제는 "이걸 계속 반복해야 한다" 는 거예요. 매주 블로그 1편·카드뉴스 1세트·달력 1장을 손으로 다시 만들어야 한다면 1인 마케팅 대표는 금방 지칩니다.

그래서 Phase 3부터는 "만드는 과정을 자동화" 하는 구간으로 넘어갑니다. 오늘 만든 포트폴리오도 Phase 3가 끝나면 매주 한 번씩 자동으로 최신화되는 구조로 연결됩니다.

  • 01🎤 3분 발표 · 평가가 아닌 공유 자리→
    팀별로 3분씩 발표합니다. 완성도를 평가하는 자리가 아니라 서로의 접근을 열어 보이는 자리입니다.
  • 02📋 네 가지만 · 사례·Before·After·화면→

    다른 팀 아이디어를 보는 것 자체가 학습입니다. 질문과 피드백은 슬랙 #수업실습 스레드에 달아 주시면 수업 후에도 이어집니다.

Phase 3 예고 · 자동화 다리

  • 01😤 손 반복의 한계 · 매주 다시 만들면 지쳐요→
    매주 블로그 1편·카드뉴스 1세트·달력 1장을 손으로 다시 만든다면 1인 마케팅 대표는 금방 지쳐요.
  • 02🏠 10강 비서 상주 · 폴더+가이드북으로 전환→
    AI에게 집 주소(폴더)와 가이드북(CLAUDE.md · AI가 읽는 프로젝트 안내서 파일)을 알려 주고 여기서 알아서 일해줘라고 말하는 단계예요. 비서를 집에 상주시키는 단계라고 보면 돼요.
  • 03🗺️ Phase 3 흐름 · 기초→자동화→에이전트→
    10강 터미널(Terminal · 명령어로 컴퓨터를 조작하는 검은 창)+CLAUDE.md(기초 도구) → 11~13강 반복 작업 자동화 → 14~15강 첫 번째 에이전트(Agent · 자율 실행 AI) 만들기 순으로 흘러요.
  • 04📁 오늘 포트폴리오 · 삭제 금지 Phase 3 재료→
    오늘 만든 포트폴리오는 삭제하지 마세요. Phase 3에서 자동 업데이트되는 포트폴리오 실습의 원본으로 다시 열립니다.
