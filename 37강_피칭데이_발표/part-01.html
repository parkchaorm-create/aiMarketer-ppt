반갑습니다. 김국희입니다. 오늘은 이 과정의 마지막 회차, 37강이에요. 34·35·36강 파이널 3부작을 거쳐 오신 팀들의 자료가 오늘 한자리에 모입니다.

오늘의 목표는 단순합니다. 지금까지 만든 것을 실제 청중 앞에서 한 번 선보이는 것. 새로 배우는 지식은 없어요. 순위를 가르는 경연도 아닙니다. 완주를 함께 축하하는 자리입니다. 결과물이 라이브에서 완벽하게 돌지 않아도 괜찮습니다. 라이브 장애는 실무 피칭(Pitch · 짧은 발표로 가치를 설명하는 것)에서도 종종 발생합니다. 대응하는 태도가 역량의 일부예요.

37회 여정 · 여섯 단계

  • 01 🎨 Phase 1~2 · 브랜드 + 콘텐츠 →
    1~4강 브랜딩 기초(아이덴티티·페르소나·팀 매뉴얼) + 5~9강 콘텐츠 파이프라인(SEO·SNS·포트폴리오). 대표 산출물은 팀 브랜드 매뉴얼과 포트폴리오 PDF·웹 시안.
  • 02 ⚙️ Phase 3~4 · 자동화 심장 →
    10~15강 로컬 오케스트레이션(CLAUDE.md · 스킬 · 자동 에이전트) + 16~25강 GHL MCP(Model Context Protocol · Claude에 외부 도구를 연결하는 프로토콜) 파이프라인.
  • 03 🎬 Phase 5~6 · 영상 + 창업 →
    26~31강 HIGGSFIELD 영상 생성 + 32~37강 창업·파이널(비즈니스 모델·세일즈·포트폴리오 웹·스케줄링·리허설·오늘 피칭). 최종 산출물은 팀 피칭 영상 + 심사 피드백.

PM 대표 · 6항목 점검

  • 01 📄 발표 슬라이드 · 지금 열기 →
    36강 리허설 자료(Google Slides·Canva·PDF)가 발표 시작 즉시 열리는 상태인지 탭·창을 미리 띄워 둡니다.
  • 02 🌐 34·35강 · 웹·GHL 준비 →
    34강 포트폴리오 URL과 35강 GHL(GoHighLevel · 올인원 마케팅 자동화 플랫폼) 대시보드 또는 스크린샷 세트가 즉시 열리는지 확인합니다.
  • 03 🎞️ 36강 · 리허설 녹화 로컬 →
    라이브 실패 시 재생용으로 36강 리허설 녹화본이 컴퓨터 로컬 폴더에 저장돼 있어야 합니다. 인터넷 없이도 재생 가능한 상태.
  • 04 🔌 전원·와이파이·역할 분담 →
    팀 컴퓨터 전원선·와이파이 확인 + 누가 말하고 누가 화면 전환·보조할지 발표자·조수 역할을 발표 전에 정해 둡니다.
