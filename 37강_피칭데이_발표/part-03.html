많은 팀이 36강 리허설에서 한 번씩 경험하셨을 거예요. "분명히 되던 게 갑자기 안 돼요." 인터넷이 느려지거나, 계정이 로그아웃되거나, API 키 만료가 그 순간에 터지거나. 흔한 일입니다. 해외 스타트업 피칭 가이드들도 "라이브 데모는 항상 백업을 준비하라" 는 조언을 공통으로 실어 둡니다[^13].

오늘은 라이브에서 실패하더라도 발표가 이어질 수 있도록 백업 3단을 같이 걸어두겠습니다. 이게 오늘 Part 03의 전부입니다.

아래 일곱 가지를 팀별 PM이 발표 시작 전에 직접 눌러 보세요. 서로 점검해 주시면 더 좋습니다.

  • 02 🛟 백업 3단 · 실패 대응 →

  • 03 🎤 실패 멘트 · 미리 외우기 →
    실패 순간에 당황하지 않도록 팀별 PM이 미리 한 문장 외워 두는 대응 멘트 3가지를 준비했습니다.

💬 그대로 읽으세요
1. 컴퓨터 전원 · 어댑터 — 배터리가 아닌 전원선 연결 상태 2. 와이파이 연결 — 강의장 SSID 접속 + 브라우저 첫 페이지 로딩 확인 3. 34강 포트폴리오 URL — 탭 열어 놓기 + 새로고침 한 번 4. 35강 GHL 대시보드 — 로그인 유지 상태 + 주요 자동화 1건 클릭 가능 5. 36강 리허설 녹화본 — 로컬 폴더에 저장되어 있음 확인 (인터넷 없이도 재생 가능) 6. 발표 슬라이드 — PDF로도 내보내 로컬 저장 (Google Slides 오프라인 대비) 7. 프로젝터·HDMI·어댑터 — 운영팀이 사전 준비 · 팀별 컴퓨터과의 호환성 확인

라이브 시연이 막히는 상황은 크게 세 종류입니다. 인터넷 장애 · 계정/권한 문제 · 도구 자체 오류. 각각에 대응하는 백업 3단을 걸어두겠습니다.

분명히 되던 게 · 왜 안 되나요?

  • 01 😰 36강 리허설 · 한 번씩 경험 →
    "분명히 되던 게 갑자기 안 돼요." 인터넷 지연·계정 로그아웃·API 키 만료가 그 순간에 터지는 경험 다들 있으시죠.
  • 02 🌍 해외 가이드 · 공통 조언 →
    해외 스타트업 피칭 가이드들도 "라이브 데모는 항상 백업을 준비하라"를 공통으로 실어 둡니다. 흔한 실무 상황입니다.
  • 03 🛟 오늘 목표 · 안전망 걸기 →
    실패하더라도 발표가 끝까지 이어지도록 백업 3단을 같이 걸어 두는 것. 이게 오늘 Part 03의 전부입니다.

팀별 PM · 7항목 직접 누르기

  • 01 🔌 전원 + 와이파이 · 필수 →
    컴퓨터 전원 어댑터(배터리 아님) + 강의장 와이파이 SSID 접속 + 브라우저 첫 페이지 로딩까지 직접 확인합니다.
  • 02 🌐 포트폴리오 + GHL · 탭 열기 →
    34강 포트폴리오 URL 탭 열어 놓고 새로고침 1회 + 35강 GHL 대시보드 로그인 유지 상태 확인. 자동화 1건은 클릭 가능해야 합니다.
  • 03 🎞️ 녹화본 + PDF · 로컬 저장 →
    36강 리허설 녹화본이 로컬 폴더(바탕화면·디스크)에 있어 인터넷 없이 재생 가능 + 발표 슬라이드 PDF 로컬 저장(Google Slides 오프라인 대비).
  • 04 📺 프로젝터 · HDMI 확인 →
    운영팀이 사전 준비 + 팀 컴퓨터와 HDMI·어댑터 호환성 확인. 슬랙 #수업실습에 "✅ 7/7 완료" 한 줄 업로드.
