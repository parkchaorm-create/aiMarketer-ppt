실습 B에서는 각 팀이 Claude Code 로 피칭 스크립트 초안 한 편(약 500단어 · 3분 분량) 을 생성하고, Google Slides 에 제목 1장 + 본문 5장 = 총 6장 을 배치합니다. 이 두 파일이 오늘의 두 번째 산출물입니다.

1. Claude Code 실행: 터미널에서 `claude` 입력 → 프로젝트 폴더로 이동. 2. 프롬프트 1개 복사·붙여넣기: 아래 R·C·E·C·F(역할·맥락·예시·제약·포맷) 구조 프롬프트. 3. 초안 수신·저장: Claude Code가 제안하는 `pitch-script.md` 파일로 저장. 4. 30초 단위 다듬기: 팀이 한 블록씩 돌아가며 실제 말로 읽어 보며 고치기. 어색한 문장은 줄이고 블록당 한 문장 핵심만 남기기. 5. Google Slides 6장 배치: slides.google.com 에서 새 프레젠테이션 → 빈 레이아웃 6장 생성 → 블록별 핵심 키워드 3개씩 배치. 6. 최종본 저장: 스크립트 `pitch-script.md` 와 슬라이드 공유 링크 모두 `#수업실습` 채널에 업로드.

R · C · E · C · F · 5층

  • 01 🎭 R · Role / 🌐 C · Context →
    R·C·E·C·F(역할·맥락·예시·제약·포맷). R은 "1인 마케팅 에이전시 피칭 작가". C는 가상 클라이언트 + 우리 4가지 산출물.
  • 02 💡 E · Example / 🚫 C · Constraint →
    E는 Hook·Our Take 한 줄 예시 제공. C는 6블록 준수 + 결과 약속·연령 호명·수익 보장 금지 규정.
  • 03 📄 F · Format · pitch-script.md →
    F는 Markdown(마크다운 · 문서 작성 표기법) 파일로 저장. 각 블록 H2 + "대본:" + "시간:" + 단어 수.

  • 01 💾 pitch-script.md · 로컬 저장 →

    각 블록에 "대본:" 본문과 "시간:" 초 단위 표기 모두 있는지. 시간 표기가 다음 Part 06 타임박싱에 쓰입니다.
  • 04 🎞️ Slides 6장 · 빈 슬라이드 없음 →
