실습 A에서는 각 팀이 선택한 가상 클라이언트 기준 으로, 5개 카테고리 × 4항목 = 총 20항목 QA 체크리스트 를 순서대로 실행하고 결과를 기록합니다. 이 체크리스트 1장이 오늘의 첫 번째 산출물입니다.

1. 팀 공유 문서(구글 닥스 또는 노션) 에 아래 체크리스트 템플릿 복사·붙여넣기. 2. 카테고리 1번(포트폴리오) 부터 순서대로 실행하며 ✓ / ✗ 표시. 3. 실패한 항목은 "증상 한 줄 + 의심 원인 한 줄" 추가. 4. 20항목 전부 끝나면 최하단 합격선 체크 한 줄 작성. 5. `#수업실습` 채널에 업로드(파일명: `[36강][클라이언트태그] 파이프라인_QA체크.md`).

복사 · 체크 · 메모 · 업로드

  • 01 📋 템플릿 복사 · 공유 문서 →
    구글 닥스 또는 노션 팀 공유 문서에 20항목 체크리스트 템플릿 복사·붙여넣기. 팀 단위 1개 문서로 통합.
  • 02 👥 3인 분담 · 한 사람당 7항목 →
    3인 팀이면 한 사람당 약 7항목씩 분담 · 병렬 진행 후 마지막에 합치기. RACI(Responsible·Accountable) 명확히.
  • 03 📝 실패 메모 · 한 줄 필수 →
    실패한 항목은 "증상 한 줄 + 의심 원인 한 줄"을 반드시 메모. 이 메모가 37강 당일 아침 수정 리스트입니다.

모두 ✓ · 필요 없다고요?

  • 01 🎯 합격선 · 4×2 · 20중 8 →
    20항목 전부 ✓일 필요 없습니다. 4카테고리 이상에서 각 2항목 이상 ✓면 실습 A 합격. 최소 8개.
  • 02 ❗ 실패 · 부끄러움 아님 →
    실패 항목 자체가 중요. "어디서 어떻게 실패했는지 한 줄 메모"가 ✓ 개수보다 가치 있습니다.
  • 03 📆 메모 = 37강 · 아침 리스트 →
    이 실패 메모가 내일 37강 피칭 데이 아침에 바로 참고할 수정 재료가 됩니다. 완벽보다 기록이 먼저.

분담 · 재활용 · 지름길

  • 01 👥 RACI · R만 한 명씩 →
    RACI(R실행·A결재·C자문·I통보)에서 항목당 R(Responsible · 실행 담당) 한 명을 분명히. 3인 팀은 7+7+6 식 분담.
  • 02 ♻️ 25강 기록표 · 재활용 →
    C3(GHL) 복잡할 때 25강에서 만든 5개 테스트 케이스 기록표 재활용 가능. 새로 돌리지 않아도 됩니다.
  • 03 📱 모바일 반응형 · F12 우회 →
    C1.2 모바일 확인 어렵다면 브라우저 개발자 도구(F12 → 모바일 아이콘)로 대체 가능. 실기기 없어도 통과.
