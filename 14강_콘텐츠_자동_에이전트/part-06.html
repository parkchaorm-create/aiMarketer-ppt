명령어 파일 · 생각보다 · 짧을까요?

  • 01 📏 예상 · 수백 줄 코드 →
    "명령어 파일"이라고 하면 수백 줄 코드를 떠올리기 쉽습니다. 프로그래밍 경험이 없으면 더 그렇고요.
  • 02 ✏️ 실제 · 10~20줄 충분 →
    실제로는 10~20줄 정도면 충분합니다. 마크다운 파일이라 읽기도 쉬워요. 역할·입력·작업 순서·출력 형식·체크리스트.
  • 03 🔒 괜찮아요 · 첫 시도 이상 →
    실행 결과가 첫 시도에 기대와 다르게 나오는 것이 정상. 생성형 AI는 확률적 결과물이라 2~3회 시도합니다.

체크리스트 · 4항목 · 선택 1개

  • 01 ✅ 체크 · 금지어 · 연령 호명 →
    출력물에 금지어가 섞였는지 재확인 + 연령 직접 호명("40대 엄마") 없는지 확인. 대체 표현("자녀를 둔 부모") 권장.
  • 02 ✅ 체크 · 수치 완충어 · 보장성 →
    수치에 완충어("대략"·"사례별 상이") 포함 여부 + 보장성 표현("절대"·"반드시 성공") 없음 확인.
  • 03 🔁 재실행 · 지침 선명화 →
    통과하지 않는 항목이 있으면 CLAUDE.md 해당 지침을 더 분명하게 고친 뒤 재실행. 흔한 원인은 모호한 "출력 형식 규칙".
  • 04 🤖 선택 · tone-checker 에이전트 →
    시간 여유 팀만. `.claude/agents/tone-checker.md`에 금지어 검수 전담 서브에이전트 1개를 맛보기로 추가.
