02 / 07

누가 무엇을 · 다시 정할까요?

  • 01 🏁 팀 프로젝트 첫 관문 →
    팀 프로젝트 첫 관문은 "누가 무엇을 하는가"를 정하는 자리. 출발선을 맞추고 시작합니다.
  • 02 🔄 4강 팀 · 역할 이름만 변경 →
    4강 팀을 그대로 유지하되 역할 이름만 바뀝니다. 4강은 "문서 작업", 14강은 "자동화 작업"이거든요.
  • 03 🎯 목적 · 팀 호흡 유지 →
    4강에서 쌓인 팀 호흡을 그대로 이어갑니다. 새로 짜지 않고 자동화 맥락에 맞게 역할만 재정의합니다.

03 / 07

네 역할 · 한 사람당 · 한 산출물

  • 01 🎤 팀 리더 · 시간·발표 →
    Lead — 시간 관리 · 합의 진행 · 3분 발표 대본 담당. 팀 전체 흐름의 조율자 역할입니다.
  • 02 📐 아키텍트 · 3W·다이어그램 →
    Architect — 3W 목적 문장 주도 · 아키텍처 다이어그램 1장 설계. 4강 Designer 계열과 호흡이 맞습니다.
  • 03 ⌨ 엔지니어 · 명령어·실행 →
    Engineer — 슬래시 명령어 파일 작성 · Claude Code에서 실행 테스트. 4강 Reviewer 계열에 맞습니다.
  • 04 📝 큐레이터 · CLAUDE.md →
    Curator — 팀 CLAUDE.md 작성 · 4강 매뉴얼 주입 담당. 4강 Editor 계열에 자연스럽게 이어집니다.

04 / 07

실무 프레임 · 팀용 · 단순화

  • 01 📋 RACI · 네 범주 역할 프레임 →
    RACI(Responsible·Accountable·Consulted·Informed · 실행·책임·의견·통지로 구분하는 팀 역할 프레임)는 업계 표준입니다.
  • 02 ✂️ 오늘 · 4역으로 단순화 →
    공공 강의 맥락에서 RACI를 그대로 쓰면 복잡합니다. 오늘은 리더·아키텍트·엔지니어·큐레이터 4역으로 축약.
  • 03 🧑‍🤝‍🧑 5명 팀 · 엔지니어 두 명 →
    팀이 5명이면 엔지니어를 두 분이 나눠 맡으셔도 좋습니다. 나머지 세 역할은 한 사람씩 고정.

05 / 07

폴더 · 세션 · 파일 · 매뉴얼 · 한 줄

  • 01 📁 단계 1~2 · 폴더 + 세션 →
    바탕화면에 `AI마케터_14강_팀{번호}` 폴더 생성 후 해당 폴더에서 `claude` 명령으로 Claude Code 세션 시작.
  • 02 📄 단계 3 · 파일 두 개 생성 →
    `CLAUDE.md` 한 개 + `.claude/commands/team-content.md` 한 개를 Claude Code에게 한 줄로 요청해 생성합니다.
  • 03 📋 단계 4 · 매뉴얼 주입 →
    큐레이터가 4강 매뉴얼 PDF에서 5개 섹션(정체성·페르소나·컬러·보이스·금기어)을 CLAUDE.md에 붙여 넣습니다.
  • 04 #️⃣ 단계 5 · 상단 한 줄 →
    CLAUDE.md 맨 위에 `# 팀 {팀명} · {클라이언트이름} 콘텐츠 생산 팀 CLAUDE.md` 한 줄을 추가합니다.
