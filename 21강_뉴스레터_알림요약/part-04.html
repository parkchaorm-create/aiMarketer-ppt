💬 그대로 읽으세요
앞에서 "나가는 목소리와 들어오는 신호를 한 쌍으로 설계한다"고 말씀드렸어요. 먼저 나가는 목소리, 즉 뉴스레터부터 만듭니다.

GHL의 이메일 빌더(Email Builder)는 드래그&드롭 방식에 AI 카피 보조(AI Assistant)가 내장되어 있어서, 본문·제목·행동 유도 문구(Call to Action · CTA)를 프롬프트 한두 줄로 초안까지 받아낼 수 있어요[^6]. 오늘은 그 기능을 클로드 코드와 하이브리드로 써 봅니다 — 클로드 코드가 한국어 본문 초안을 짜고, GHL 빌더가 레이아웃·링크를 마무리하는 식이에요.

준비물 체크 4가지

  • 01 🏷️ 17강 태그 · 5종 살아 있음 →
    `need-nurturing`·`high-intent`·`vip-candidate`·`budget-low`·`cold` 태그가 17강 기준 그대로 있는지 확인합니다.
  • 02 🎭 가상 클라이언트 · A/B/C/D →
    1강에서 고른 클라이언트(푸르트·슬로우템포·손결·링크드마인드) 중 하나로 오늘 뉴스레터 톤과 소재를 맞춥니다.
  • 03 📧 GHL 이메일 빌더 · 로그인 →
    Marketing → Email → Campaigns 메뉴 접근 가능 상태 + 본인 수신용 이메일 주소 하나를 준비합니다.
  • 04 💬 Slack 워크스페이스 · 권한 확인 →
    관리자 권한이 있으면 그대로, 없으면 수업 중 개인 워크스페이스 무료 생성. 앱 설치 권한이 핵심입니다.

📝 강사 노트
💬 그대로 읽으세요
세부 단계는 노션 튜토리얼(`실습_튜토리얼/실습A_뉴스레터.md`)에 화면 캡처와 함께 정리됩니다. 여기서는 큰 지도만 보고 가겠습니다.

1. 가상 클라이언트 확인 — 1강에서 고른 A/B/C/D 중 하나. 뉴스레터의 톤·소재가 이 클라이언트 기준이에요. 2. 클로드 코드로 본문 초안 생성 — 아래 복사용 프롬프트 블록 참조. 결과는 초안이며 GHL 빌더에 붙일 재료로 씁니다. 3. GHL Email Campaign 신규 생성 — Marketing → Email → Campaigns → New Campaign. Regular Campaign 선택. 4. 템플릿 선택 + 본문 붙여넣기 — Blank 또는 기본 템플릿에서 시작. 클로드 코드 초안을 본문에 붙이고, 제목 라인을 작성. 5. 수신자 세그먼트 지정 — `need-nurturing` OR `high-intent` 태그 보유자 지정. `budget-low` 제외 필터 추가. 6. 발송 일정 설정 — 우선 즉시 발송(본인 수신자 1명) 테스트 → 확인 후 매월 첫째 주 월요일 오전 10시 예약. 7. 본인 수신 확인 — 받은편지함에서 뉴스레터 도착 확인 → 슬랙 `#수업실습` 채널에 `[21강][A/B/C/D] 뉴스레터수신화면` 스크린샷 업로드.

  • 01 🏷️ need-nurturing · 정보 수집 중 →

메뉴 경로 · 권한 · 분기

  • 01 📧 GHL · Marketing → Email → Campaigns →
    실습 A(뉴스레터) 경로. 좌측 메뉴 Marketing에서 Email → Campaigns 순서. 미리 열어 두면 진입 속도가 달라집니다.
  • 02 🔀 GHL · Automation → Workflows →
    실습 B(Slack 알림) 경로. 좌측 메뉴 Automation → Workflows. 오늘 Outbound Webhook을 여기 Action 안에 넣습니다.
  • 03 🔑 Slack · 앱 설치 권한 확인 →
    기존 조직 Slack에서 앱 추가가 막힌다면 관리자 권한 부족. 이 경우 개인 워크스페이스를 새로 만들면 됩니다(5분).
  • 04 🔀 분기 · 조직 vs 개인 WS →
    조직 WS에 권한 있음 → 바로 Part 06 진입. 권한 없음 → 개인 WS 무료 생성 → Part 06. 결과는 동일합니다.

📝 강사 노트
💬 그대로 읽으세요
1. 강사가 전면 화면에서 7단계 중 핵심 구간을 시연합니다 (캠페인 생성 → 세그먼트 지정 → 스케줄 저장 → 테스트 발송). 2. 시연 후 수강생은 각자 자리에서 본인 가상 클라이언트용 뉴스레터를 세팅합니다. 3. 테스트 발송은 본인 이메일 1개 수신자만으로 제한. 외부 대량 발송 금지. 4. 완성된 수강생은 슬랙 `#수업실습` 채널에 `[21강][A/B/C/D] 뉴스레터수신화면` 형식으로 수신 스크린샷을 업로드. 5. 막히는 분은 `#강사-QnA` 채널 또는 손 들기로 강사·운영진 호출.

오늘 우리는 뉴스레터를 직접 세팅했고, 세그먼트와 스케줄까지 지정했습니다.
