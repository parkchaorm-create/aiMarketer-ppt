두 웹훅 · 정말 같은 URL 하나인가요?

  • 01 🔗 한 URL · 두 역할 →
    Slack이 발급한 Incoming Webhook URL 한 개 = GHL Outbound Webhook 목적지. 같은 URL을 양 끝에서 다르게 부를 뿐입니다.
  • 02 ⚠️ 한 번에 성공 · 오히려 드뭄 →
    새로 만지는 영역이 많아 두세 번 되돌아가는 것이 자연스러운 회차예요. 막히면 Slack 쪽부터 되짚으면 대부분 풀립니다.
  • 03 🔒 URL · 비밀번호급 민감 정보 →
    Webhook URL은 공개 채팅·저장소에 붙이지 마세요. 스크린샷에도 포함되지 않도록 가리고 업로드합니다.

03 / 07

Slack · 워크스페이스 → URL

  • 01 1️⃣ WS 확인 · 조직 or 개인 →
    관리자 권한이 있는 워크스페이스라면 그대로, 없으면 개인 워크스페이스 무료 생성. 5분이면 충분합니다.
  • 02 2️⃣ 채널 생성 · #lead-alerts →
    전용 알림 수신 채널을 새로 만듭니다. 테스트 중 다른 대화와 섞이지 않게 하기 위한 기본 세팅입니다.
  • 03 3️⃣ Slack 앱 생성 · From scratch →
    api.slack.com/apps → Create New App → From scratch → 앱 이름·워크스페이스 선택. Slack 공식 절차예요.
  • 04 4️⃣ Incoming Webhook · 활성화 →
    Features → Incoming Webhooks → Activate 토글 On → Add New Webhook → 2단계 채널 선택. 공식 수신 URL이 발급됩니다.

04 / 07

GHL · Workflow → Webhook

  • 01 6️⃣ Workflow 신규 · Create from Scratch →
    Automation → Workflows → New Workflow → Create from Scratch. 이름 예시는 "VIP 태그 Slack 알림"이에요.
  • 02 7️⃣ 트리거 · Contact Tag · vip-candidate →
    Add Trigger → Contact Tag → "Tag Added" → 태그 `vip-candidate` 선택. 태그 추가 순간이 트리거 조건입니다.
  • 03 8️⃣ 액션 · Outbound Webhook + JSON →
    Method POST · URL에 Slack 수신 URL 붙여넣기 · Headers `Content-Type: application/json` · Body에 `{{contact.XXX}}` 변수 JSON.
  • 04 9️⃣ Publish + 테스트 · 가상 제출 →
    워크플로우 Publish 후 17강 폼에 `vip-candidate` 조건이 되는 가상 데이터 1건 제출 → Slack 채널 알림 확인.

CONCEPT · 포맷 + 실패 시
05 / 07

4~6줄 요약 · 실패 시 4체크

  • 01 📝 메시지 포맷 · 4~6줄이 실무 표준 →
    이모지 제목 1줄 + 이름·이메일·관심·가능 시간 4줄 + 링크 1줄. 길면 읽히지 않아요. 짧고 선명하게 만듭니다.
  • 02 🚫 실패 ① · URL 오타 · https:// 누락 →
    Outbound Webhook URL 맨 앞 `https://` 누락이 가장 빈발 실패 원인. 복사할 때 한 글자씩 점검하세요.
  • 03 🚫 실패 ② · Published 아님 · 태그 미부착 →
    GHL 워크플로우가 Published 상태인지 + 테스트 리드에 실제 `vip-candidate`가 붙었는지 17강 워크플로우부터 되짚으세요.
  • 04 🤝 옆자리 교차 테스트 · 업로드 →
    옆자리와 서로의 폼에 가상 이메일 제출. 본인 Slack에 상대방 이름이 뜨면 성공 → 스크린샷(URL 가리고) 업로드.

06 / 07

Part 06 · 네 줄 정리

  • 01 🔗 구조 · Workflow → Webhook → Slack →
    GHL Workflow의 Outbound Webhook 액션 → Slack Webhook URL → Slack 채널. 한 URL을 양 끝에서 부르는 파이프입니다.
  • 02 🎯 트리거·동작 · Tag Added → POST JSON →
    트리거는 `Tag Added: vip-candidate`, 동작은 POST + JSON 페이로드. Merge Field는 17강 폼 필드명과 동일해야 해요.
  • 03 🔒 URL 보안 · 공개 금지 · 스크린샷 주의 →
    Slack Webhook URL은 비밀번호급 민감 정보. 채팅·저장소·스크린샷 모두에서 노출되지 않도록 관리합니다.
  • 04 📤 합격선 · Slack 알림 스크린샷 →
    `#수업실습` 채널에 `[21강][A/B/C/D] Slack알림수신화면`으로 URL 가린 스크린샷 업로드가 실습 B 합격선입니다.
