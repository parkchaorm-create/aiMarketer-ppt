실습 A · 3덩어리 구조

  • 01 📝 A부 · 초안 · 클로드 코드 →
    클로드 코드에 복사용 프롬프트를 붙여 넣고 월간 뉴스레터 본문 초안을 받습니다. 이후 GHL 빌더에 붙일 재료입니다.
  • 02 🏗️ B부 · 빌드 · GHL Campaign →
    New Campaign → 템플릿 선택 → 본문 붙여넣기 → 수신자 세그먼트 지정 → 발송 일정. 단계 3~6에 해당해요.
  • 03 📦 C부 · 확인 · 본인 수신 →
    테스트 발송(본인 1명) → 받은편지함 확인 → `#수업실습` 채널 스크린샷 업로드까지가 합격선입니다.

📝 강사 노트
💬 그대로 읽으세요
1. Slack 워크스페이스 확인 — 본인 워크스페이스 관리자 권한이 있으신 분은 그대로, 없으신 분은 수업 중 개인 워크스페이스 새로 생성 (무료). 2. 알림 수신용 채널 생성 — 예: `#lead-alerts`. 테스트 중 혼란 방지를 위해 전용 채널 권장. 3. Slack 앱 생성 — https://api.slack.com/apps 접속 → Create New App → From scratch → 앱 이름·워크스페이스 선택[^7]. 4. Incoming Webhook 활성화 — 좌측 메뉴 Features → Incoming Webhooks → Activate Incoming Webhooks 토글 On → Add New Webhook to Workspace → 2단계에서 만든 채널 선택[^8]. 5. Webhook URL 복사 · 안전 보관 — 생성된 URL(`https://hooks.slack.com/services/T.../B.../xxx`)을 메모장에 임시 저장. 이 URL은 비밀번호와 같은 수준의 민감 정보이므로 `#수업실습` 채널 등 공개 공간에 붙여 넣지 마세요[^9].

6. GHL Workflow 신규 생성 — Automation → Workflows → New Workflow → Create from Scratch. 이름 예: "VIP 태그 Slack 알림". 7. 트리거 설정 — Add Trigger → Contact Tag 트리거 → "Tag Added" → 태그 선택 `vip-candidate`[^10]. 8. 액션 설정 · Outbound Webhook — Add Action → Webhook (Outbound).

Method: `POST`

  • 01 1️⃣ Step 1~2 · 클라이언트 + 초안 →

    테스트 즉시 발송(본인 1명) → 수신 확인 → 매월 첫째 주 월요일 10시 예약 → 스크린샷 업로드까지.

GHL의 변수(Merge Field)는 `{{contact.XXX}}` 형식으로 자동 치환돼요. 필드명은 17강에서 만든 폼 필드와 동일하게 맞추세요.

9. 워크플로우 활성화(Publish) + 테스트 제출 — Publish → 17강에서 만든 폼에 `vip-candidate` 조건이 되도록 가상 데이터 1건 제출 → Slack 채널에 알림이 올라오는지 확인.

실무에서 내부 알림 메시지는 4~6줄 이내가 권장돼요. 길면 오히려 읽히지 않습니다. 추천 포맷:

프롬프트 요건 · 자주 막히는 곳

  • 01 📋 프롬프트 · 500자 · 담백·차분 →
    클라이언트·대상·분량(500자)·톤(담백)·구성(제목·인사·소식·추천·맺음)·금지(수익 약속·재촉) 6개 축으로 지정합니다.
  • 02 🚫 세그 0명 · 태그 오타 점검 →
    수신자 0명으로 뜬다면 태그 철자 오타 또는 17강 태그 미생성. Contacts → Smart Lists에서 보유자 수부터 확인하세요.
  • 03 📭 스팸 분류 · 5~10분 대기 →
    테스트 발송이 안 올 때 스팸함 우선 확인. 이메일 도메인 미인증 상태라 받은편지함 대신 스팸에 분류되는 경우가 많아요.
  • 04 ♻️ 월간 반복 · 수업은 1회만 →
    단일 Campaign은 1회 발송 단위. 월간 반복은 복제 또는 별도 워크플로우 경로가 필요해 오늘 수업에서는 1회 + 예약까지만 합니다.

시연 → 따라 하기 → 업로드

  • 01 🎥 1) 시연 · 7단계 핵심 구간 →
    강사가 전면 화면에서 캠페인 생성 → 세그먼트 지정 → 스케줄 저장 → 테스트 발송 순서로 먼저 보여 드립니다.
  • 02 🧑‍💻 2) 수강생 · 본인 자리에서 →
    시연 후 각자 자리에서 본인 가상 클라이언트용 뉴스레터를 세팅합니다. 막히면 손 들어 강사·운영진 호출.
  • 03 ✉️ 3) 테스트 · 본인 이메일 1명만 →
    테스트 발송은 본인 이메일 1개 수신자로만 제한합니다. 외부 대량 발송은 수업 규정상 금지예요.
  • 04 📤 4) 업로드 · #수업실습 채널 →
    `[21강][A/B/C/D] 뉴스레터수신화면` 포맷으로 본인 수신 스크린샷 업로드. 여기까지가 실습 A 합격선입니다.
CONCEPT · 03
📝 강사 노트
💬 그대로 읽으세요
Slack은 `text` 대신 Block Kit 형식의 풍부한 레이아웃도 지원합니다[^11]. 버튼·이미지·구분선까지 넣을 수 있지만 오늘은 `text` 한 줄짜리 기본 포맷으로 충분합니다. 실무에서 점점 확장하실 수 있어요.

GHL에는 Slack Premium Action도 내장되어 있어 Outbound Webhook 없이 바로 Slack 채널에 보낼 수 있어요[^12]. 다만 플랜에 따라 프리미엄 활성화 여부가 다르고, 수업 환경에서는 Outbound Webhook 주경로가 모든 플랜 호환이라 오늘은 이 경로로 진행합니다.

1. 강사가 Slack 앱 만드는 A 구간 5단계를 전면에서 시연합니다. 2. 수강생이 A 구간을 끝내면 손 들어 확인 → 강사가 URL 복사 여부만 점검. 3. B 구간은 강사가 1개 워크플로우를 처음부터 끝까지 시연한 뒤 수강생이 따라 만듭니다. 4. 완성된 수강생끼리 옆자리와 서로의 폼에 테스트 제출 — 본인 Slack에 상대방이 넣은 이름이 뜨면 성공. 안 되면 본인이 본인 폼에 제출. 5. 성공한 수강생은 슬랙 `#수업실습` 채널에 `[21강][A/B/C/D] Slack알림수신화면` 형식으로 Slack 알림이 울린 스크린샷만 업로드. Webhook URL은 스크린샷에 포함되지 않도록 주의.

Slack Webhook URL은 공개 금지. 슬랙 `#수업실습`에 올릴 스크린샷은 "알림이 울린 Slack 채널 화면"만 찍어 주세요.
