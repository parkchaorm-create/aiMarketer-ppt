리포트는 · 숫자 나열 아니시죠?

  • 01 📉 흔한 오해 · 엑셀 빼곡한 지표 →
    "성과 리포트" 하면 KPI(Key Performance Indicator · 핵심 성과 지표)가 빼곡한 엑셀이 먼저 떠오르세요. 이게 흔한 출발점입니다.
  • 02 🧭 본질 · 다음 주 방향 결정 문서 →
    실무 리포트의 본질은 "지난 기간 숫자 정리 + 다음 기간에 무엇을 할지 결정"입니다. 에이전시 월간 리포트가 정확히 이 형태입니다.
  • 03 ✍️ 관건 · 다음 액션 한 줄 →
    4블록 중 "다음 액션"이 없으면 리포트는 읽고 끝입니다. 액션이 있어야 다음 주 기획이 됩니다.

Markdown 한 장 · 슬래시 한 줄

  • 01 📄 .claude/commands/report.md · 한 장 →
    `.claude/commands/` 폴더에 Markdown 파일 한 장을 두면 그대로 슬래시 명령어가 됩니다. 복잡한 코드는 필요 없습니다.
  • 02 📂 폴더 구조 · 4개만 기억 →
    CLAUDE.md(팀 맥락) · `.claude/commands/report.md`(지시문) · `data/*.csv`(더미 데이터) · `reports/`(결과 저장).
  • 03 ⚡ /report · 한 줄 실행 →
    터미널에서 `/report`만 치면 CSV를 읽고, 팀 맥락을 반영해, 4블록 리포트 Markdown을 `reports/` 폴더에 저장합니다.

CONCEPT · 수동 vs 자동
05 / 07

자동 = 사람 불필요 · 아닙니다

  • 01 ⏰ 소요 · 1~2시간 → 30초~1분 →
    수동 작성은 경험상 사례별 1~2시간. 자동 생성은 30초~1분 대기 후 결과물이 나옵니다.
  • 02 📏 편차 · 들쭉날쭉 → 4블록 일관 →
    수동은 주마다 형식이 달라지기 쉽지만, 자동은 4블록 구조가 항상 일관되게 유지됩니다.
  • 03 🔍 검수 · 여전히 대표의 몫 →
    도구는 "구조 맞춰 초안을 내는 일"만 대신합니다. 숫자 해석과 액션 판단은 여전히 대표가 합니다.
CONCEPT · 03
📝 강사 노트
💬 그대로 읽으세요
원칙: "무엇을 · 언제 · 누가"까지 구체적으로

Claude Code는 `.claude/commands/` 폴더에 Markdown 파일 한 장을 두면 슬래시 명령어가 됩니다[^4]. 예를 들어 `.claude/commands/report.md` 파일을 만들고 그 안에 "CSV 두 개를 읽어서 4블록 구조로 Markdown 리포트를 써라"는 지시문을 적어두면, 터미널에서 `/report`만 입력해도 전 과정이 실행됩니다.

내 프로젝트 폴더/ ├── CLAUDE.md ← 팀 클라이언트 정보 + 지시 ├── .claude/ │ └── commands/ │ └── report.md ← /report 슬래시 명령어 정의 ├── data/ │ ├── sample_ga.csv ← 오늘 배포되는 더미 데이터 │ └── sample_sc.csv └── reports/ ← 자동 생성된 리포트 저장 위치
