실습 A의 목표는 단 하나입니다 — 내 계정에 T2V 영상 한 편을 남긴다. 완성도는 평가하지 않습니다. 오늘 여러분이 쓴 4요소 프롬프트가 어떤 모습의 영상이 되는지, 그 과정을 손끝으로 경험하시는 게 전부예요.

1. 클라이언트 선택 · 선택한 가상 클라이언트(A/B/C/D) 중 하나의 한 장면을 상상 2. 프롬프트 작성 · Part 03의 4요소 템플릿에 맞춰 4~5문장 영어 프롬프트 작성 (한국어 번역기 활용 가능) 3. 힉스필드 입력 · Create → Text to Video 메뉴 진입 → 프롬프트 붙여넣기 4. Generate 클릭 · 대기 · 20~60초 대기 후 결과 확인 5. 결과 저장 + 업로드 · 다운로드 → 슬랙 `#수업실습` 채널에 `[26강][클라이언트태그]` 형식으로 업로드

📝 강사 노트
💬 그대로 읽으세요
움직임이 너무 빠르다/느리다 → 카메라 움직임에 "slow" 또는 "very slow" 추가

BRIDGE
📝 강사 노트
💬 그대로 읽으세요
강사와 강사가 돌아가며 자리를 둘러봅니다. 이 회차에서 가장 많이 나오는 질문 세 가지를 미리 공유드립니다.

1. "Generate 버튼 눌렀는데 아무 일도 안 일어나요" → 화면 상단에 Processing(처리 중) 또는 유사 표시가 있는지 확인. 새로고침 금지 (재생성되면 크레딧 추가 소모). 2. "결과가 제가 쓴 프롬프트랑 전혀 달라요" → 확률적 결과물이라 정상. 프롬프트 주인공(Subject) 부분을 가장 구체적으로 바꿔 한 번 재시도. 3. "영상이 너무 짧아요 (5초 정도)" → 무료 플랜 기본 길이입니다. 오늘은 길이 제어는 다루지 않습니다. 28·29강에서 다뤄요.

첫 T2V 영상 한 편을 내 계정에 남긴다 — 오늘의 합격선.
