완벽 · 합의 중 어느 쪽일까요?

  • 01 🎯 단 하나의 목표 · HD 확정 →
    실습 A는 새 영상을 만드는 시간이 아닙니다. 30강 영상을 한 단계 더 끌어올려 팀이 합의한 최종본을 정하는 시간이에요.
  • 02 🤝 완벽 ≠ 합의 · 후자가 합격선 →
    "완벽한 영상"이 아니라 "팀이 합의한 최종본"이 합격 기준. 30강의 완성도보다 합의 원칙을 그대로 이어갑니다.
  • 03 📦 34강 포폴 입력 · 미리보기 →
    오늘 확정한 HD 영상이 Phase 6의 34강 포트폴리오 웹사이트(One Page · 한 페이지 사이트)에 그대로 들어갑니다.

원본 → 슬랙 업로드

  • 01 📁 ① 팀 영상 원본 · 준비 →
    30강에서 만든 팀 영상을 팀 대표 PC 다운로드 폴더에 준비. 30강 슬랙 #수업실습 거슬러 올라가면 있습니다.
  • 02 📋 ② 체크리스트 · ✅·⚠️·❌ →
    Part 02의 10항목을 팀당 1장 점검표에 표기. 각 항목 옆에 한 줄 코멘트도 함께 적습니다.
  • 03 🔀 ③ 분기 결정 · 2개 vs 3개 →
    ⚠️·❌ 2개 이하 → 업스케일 진행 · 3개 이상 → 원본 그대로 확정 + 한계 기록. 둘 다 학습입니다.
  • 04 🎯 ④⑤ Theia · 업로드+Generate →
    higgsfield.ai/upscale → Theia 선택 → 원본 업로드 → 옵션 기본값 → Generate 클릭. 시연에서 본 그대로입니다.
  • 05 ⏳ ⑥ 대기 중 · 병렬 점검 →
    처리 대기 시간 동안 다음 팀 체크리스트 작업을 병렬 진행. 한 사람만 묶이지 않도록 RACI 역할 분담을 활용합니다.
  • 06 📤 ⑦ 결과 · 슬랙 업로드 →
    Before/After 확인 → 다운로드 → 슬랙 #수업실습 채널에 업로드. 파일명 [31강][클라이언트태그] 형식 엄수.

04 / 07

병렬 처리 · 시간 절약

  • 01 🎯 RACI · 4강에서 배운 그 표 →
    RACI(Responsible·Accountable·Consulted·Informed · 책임·실행·자문·정보)는 누가 무엇을 하는지 명확히 하는 역할 표입니다.
  • 02 👤 역할 1 · 업스케일 담당 →
    팀 대표 PC에서 업로드 · Generate · 대기 · 다운로드 진행. 한 명이 PC 앞에 앉아 Theia 작업을 끝까지 봅니다.
  • 03 👤 역할 2 · 체크리스트 담당 →
    대기 동안 원본 영상을 재생하면서 10항목 점검표 작성. ✅·⚠️·❌ + 한 줄 코멘트. 팀당 1장.
  • 04 👤 역할 3 · 파일명·업로드 담당 →
    결과 파일 이름 형식 맞추기 + 슬랙 #수업실습 업로드 + 100MB 초과 시 유튜브 비공개·구글 드라이브 대체 검토.
