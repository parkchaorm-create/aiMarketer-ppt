카드 3장 · 한 문장 · 정리

  • 01 📱 카드 1 · Social Planner →
    Social Planner(소셜 플래너 · GHL의 다채널 SNS 예약 캘린더)가 맡는 역할과 상태 5종(Save·Post·Schedule·Approval·Recurring)을 한 줄로 정리합니다.
  • 02 📰 카드 2 · Blog 롱폼 모듈 →
    Blog(블로그 · 롱폼 콘텐츠 모듈)가 Social Planner와 어떻게 다른지 · 상태 3종(Draft·Publish·Schedule)을 짚습니다.
  • 03 🔌 카드 3 · MCP 초안 엔진 →
    MCP(Model Context Protocol · AI와 외부 도구 연결 어댑터)가 "초안"은 맡고 "확정"은 사람이 맡는 하이브리드 경계를 그립니다.

02 / 07

"3채널 동시"가 · 진짜 1번인가요?

  • 01 🤔 같은 GHL · 모듈은 따로 →
    Social Planner와 Blog는 같은 GHL 안에 있지만 모듈이 달라요. 그래서 예약 액션은 두 번으로 나뉩니다.
  • 02 🧩 커버 채널 3개 · 액션 2번 →
    인스타+페이스북은 Social Planner 1회, 블로그는 Blog 1회. 합쳐서 액션 2번으로 채널 3개를 커버합니다.
  • 03 🗝️ 이 구조 · 실습의 열쇠 →
    이 구조를 먼저 알아 두면 Part 05 실습에서 "왜 저장을 두 번 하지?" 같은 혼란이 생기지 않습니다.

CONCEPT · 카드 1 · Social Planner
03 / 07

SNS 예약 캘린더 · 한 화면

  • 01 🌐 다채널 단일 예약 · 1건 → N곳 →
    페이스북·인스타·링크드인·틱톡·GBP·유튜브·블루스카이까지 하나의 캘린더에서 예약·발행·관리합니다. 게시물 1건을 여러 채널에 동시 예약.
  • 02 🎛️ 채널별 커스터마이즈 · 각 방 다른 옷 →
    같은 게시물도 인스타에는 긴 캡션 · 페이스북에는 링크 포함 짧은 텍스트. "Customize for Each Post" 토글 하나로 채널마다 다듬습니다.
  • 03 🎯 상태 5종 · 오늘은 Schedule →
    Save for Later(저장만) · Post Now(즉시) · Schedule for Later(미래 예약) · Send for Approval(승인 요청) · Recurring(반복). 오늘은 Schedule.

CONCEPT · 카드 2 · Blog
04 / 07

Blog · 롱폼 · 별도 예약 액션

  • 01 📚 Sites · Blogs · 롱폼 전용 →
    Sites 메뉴 안의 별도 모듈. 제목·부제·본문·이미지·SEO 메타까지 갖춘 롱폼을 GHL 내부에서 만들고 배포합니다.
  • 02 🎯 상태 3종 · Draft / Publish / Schedule →
    Draft(초안 · 저장만) · Publish(즉시 게시 · 과거 날짜도 가능) · Schedule(미래 예약 · 지정 시각 자동 게시). 오늘은 Schedule.
  • 03 ⚠️ Planner와 분리 · 예약 액션 2번 →
    Blog는 Social Planner와 모듈이 달라서 예약은 따로 합니다. 실습 A·B에서 "왜 저장이 두 번?"의 답은 여기에 있어요.

CONCEPT · 카드 3 · MCP
05 / 07

MCP · 초안은 AI · 확정은 사람

  • 01 🔌 MCP · 36개 공식 도구 어댑터 →
    GHL 공식 MCP 서버는 36개 도구 제공. 오늘 관련은 Social Media Posting 6개(Create·Edit·Get·Accounts·Stats 등) + Blogs 6개.
  • 02 ⚖️ 경계 · 공식 동작 명시 안 됨 →
    MCP "Create Post"가 Social Planner 예약 슬롯에 바로 꽂히는지는 공식 문서가 세부를 명시하지 않았어요. 안전한 하이브리드로 갑니다.
  • 03 🤝 역할 분담 · 재료 AI · 슬롯 사람 →
    클로드 코드 MCP = 카피·캡션·해시태그·블로그 본문 초안 엔진. GHL UI = 예약 슬롯 확정 무대. 초안은 AI · 확정은 사람이 마무리.

06 / 07

3축 · 한 문장으로 · 합치기

  • 01 🎼 한 문장 · 오늘의 공식 →
    "클로드 코드 MCP로 재료를 받아 · GHL Social Planner와 Blog에 예약을 확정한다." 이 한 문장에 3축이 다 들어갑니다.
  • 02 🧮 숫자 요약 · 2 + 3 →
    예약 액션 2번(Planner 1 + Blog 1) · 커버 채널 3개. 이 숫자 한 쌍이 실습의 나침반입니다.
  • 03 💎 가치 · 시간·일관성·확장성 →
    세팅 한 번으로 시간이 줄고 · 3채널에 톤이 일관되며 · 19강 DM·20강 재활성 회차에 그대로 이어 붙일 수 있습니다.
