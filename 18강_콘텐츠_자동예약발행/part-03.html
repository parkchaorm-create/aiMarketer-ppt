GHL 안에서 오늘 우리가 다룰 도구는 세 개입니다. 각각 무엇을 맡는지 한 줄로 정리해 볼게요.

Social Planner는 페이스북·인스타그램·링크드인·틱톡·구글 비즈니스 프로필·유튜브·블루스카이까지 주요 SNS 채널을 하나의 캘린더에서 예약·발행·관리하는 GHL 내장 모듈입니다[^1]. 핵심 특징 세 가지만 보면:

다채널 단일 예약: 게시물 1건을 여러 채널에 한 번에 예약할 수 있어요.

채널별 커스터마이즈: 같은 게시물이라도 인스타그램에는 긴 캡션, 페이스북에는 링크 포함 짧은 텍스트처럼 채널마다 다르게 다듬을 수 있습니다[^2]. "Customize for Each Post" 기능이라는 이름으로 제공돼요.

  • 01 📱 카드 1 · Social Planner →

  • 02 📰 카드 2 · Blog 롱폼 모듈 →
    Blog(블로그 · 롱폼 콘텐츠 모듈)가 Social Planner와 어떻게 다른지 · 상태 3종(Draft·Publish·Schedule)을 짚습니다.
  • 03 🔌 카드 3 · MCP 초안 엔진 →
    MCP(Model Context Protocol · AI와 외부 도구 연결 어댑터)가 "초안"은 맡고 "확정"은 사람이 맡는 하이브리드 경계를 그립니다.

📝 강사 노트
💬 그대로 읽으세요
상태 5가지: 저장만(Save for Later) · 즉시 게시(Post Now) · 미래 예약(Schedule for Later) · 승인 요청(Send for Approval) · 반복 게시(Recurring) 중 상황에 맞게 선택합니다[^3].

오늘 우리가 쓸 상태는 Schedule for Later — 미래 예약이에요. 수업 종료 30분~1시간 뒤 시점으로 예약을 잡아 보겠습니다.

GHL Blog는 Sites 메뉴 안에 있는 별도 모듈로, 제목·부제·본문·이미지·SEO 메타까지 갖춘 롱폼 콘텐츠를 직접 GHL 안에서 만들고 배포하는 공간입니다[^4]. 블로그 게시물도 예약이 가능한데, 상태가 Social Planner와 조금 달라요:

Draft (초안): 저장만 해 두고 공개 안 함

MCP · 초안은 AI · 확정은 사람

  • 01 🔌 MCP · 36개 공식 도구 어댑터 →
    GHL 공식 MCP 서버는 36개 도구 제공. 오늘 관련은 Social Media Posting 6개(Create·Edit·Get·Accounts·Stats 등) + Blogs 6개.
  • 02 ⚖️ 경계 · 공식 동작 명시 안 됨 →
    MCP "Create Post"가 Social Planner 예약 슬롯에 바로 꽂히는지는 공식 문서가 세부를 명시하지 않았어요. 안전한 하이브리드로 갑니다.
  • 03 🤝 역할 분담 · 재료 AI · 슬롯 사람 →
    클로드 코드 MCP = 카피·캡션·해시태그·블로그 본문 초안 엔진. GHL UI = 예약 슬롯 확정 무대. 초안은 AI · 확정은 사람이 마무리.

💬 그대로 읽으세요
시간 절약: 한 번 세팅해 두면 "글 쓰기·채널 맞춤 변형·예약"이 한 자리에서 끝나, 도구 사이를 오가는 피로가 크게 줄어요. 구체 감소량은 운영 상황에 따라 다릅니다.

일관성: 브랜드 톤이 3채널에 동시에 적용돼, 고객이 어느 채널에서 봐도 같은 인상을 받게 됩니다.

확장성: 오늘 만든 구조는 19강 DM·캘린더 연동, 20강 Reactivation 캠페인에 그대로 이어 붙일 수 있어요.
