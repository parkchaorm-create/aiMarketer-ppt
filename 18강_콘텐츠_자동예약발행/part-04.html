앞에서 "콘텐츠 1건을 채널에 맞춰 변형한 뒤 예약한다"는 구조를 확인했어요. 그러려면 두 가지가 먼저 준비되어야 합니다.

1. 채널 계정 연결 — Social Planner에 페이스북 페이지 + 인스타그램 비즈니스 계정이 OAuth(외부 서비스 연결 인증)로 연결되어 있어야 해요. 2. 콘텐츠 재료 — 7·8강에서 만든 자산 중 1건 + 채널별 변형안.

실습 A는 4단계로 진행합니다. 세부 단계는 노션 튜토리얼(`실습_튜토리얼/실습A_채널연결_콘텐츠준비.md`)에 화면 캡처와 함께 정리되어 있어요.

1. 가상 클라이언트 확인 — 1강에서 선택한 A(푸르트)/B(슬로우템포)/C(손결)/D(링크드마인드) 중 하나. 오늘 올릴 콘텐츠 톤은 그 브랜드에 맞춥니다. 2. Social Planner에 페이스북·인스타그램 계정 연결 — Marketing → Social Planner → Settings → 연결 추가. OAuth 플로우로 외부 로그인 + 권한 승인[^1]. 3. 7·8강 자산 중 1건 선택 — 7강 카드뉴스 이미지 또는 8강 SNS 텍스트 중 오늘 올릴 것 1건. 없으신 분은 클로드 코드로 즉석 초안 생성 (아래 대체 프롬프트 참조). 4. 채널별 변형 3벌 만들기 — 인스타그램 캡션 150~300자 + 해시태그 5~10개 / 페이스북 텍스트 200~400자 + 링크 / 블로그 제목+부제+본문 800~1500자 + SEO 메타. 클로드 코드에 프롬프트 1회 넣으면 3벌 초안이 한 번에 나옵니다.

같은 글을 3채널에 그대로 복붙하면 안 되는 이유를 실무 기준으로 정리하면 이렇습니다.

Part 04 · 네 줄 · 남기기

  • 01 🔗 연결 · OAuth 또는 시뮬레이션 →
    OAuth 플로우로 페이스북·인스타그램 연결. 안 되면 시뮬레이션 경로(Draft)로 대체 가능.
  • 02 ✂️ 변형 · 복붙 금지 · 의무 →
    채널별 변형은 의무. 길이·해시태그·링크를 채널 호흡에 맞춰 다듬습니다.
  • 03 🧾 프롬프트 1회 · 3벌 확보 →
    클로드 코드 프롬프트 1회로 3채널 초안 3벌 확보. 그 후 브랜드 톤 최종 확인.
  • 04 ♻️ 자산 재활용 · 별도 프롬프트 →
    7·8강 자산 보유자는 재활용 프롬프트 별도 제공. 원본 붙여넣기만 하면 3채널 변형 결과 반환.

📝 강사 노트
💬 그대로 읽으세요
출력 형식은 앞 프롬프트와 동일: 1. 인스타그램 캡션 (150~300자 + 해시태그 5~10개) 2. 페이스북 텍스트 (200~400자 + 링크 1개) 3. 블로그 게시물 (제목·부제·본문 800~1500자·SEO 메타)

1. 강사가 전면 화면에서 채널 연결(페이스북 페이지 1개 + 인스타그램 비즈니스 계정 1개) OAuth 흐름을 시연합니다. 2. 시연 후 수강생은 각자 자리에서 본인 테스트 페이지를 연결하거나, 연결이 어려운 경우 시뮬레이션 모드로 전환(강사 안내). 3. 클로드 코드 프롬프트 1회 실행 → 3채널 초안 결과를 로컬에 복사해 둡니다. 4. 가상 클라이언트 톤에 맞게 한 번 손을 대서 다듬습니다. 여기까지가 실습 A 완료 지점.

실 페이스북·인스타그램 계정 연결이 어려운 분은 아래 경로로 산출물에 도달하실 수 있습니다.

Social Planner: 연결된 계정이 없어도 게시물 Draft 작성까지는 가능. Part 05에서 "Save for Later" 상태로 저장 → 예약 달력에는 회색 Draft 블록으로 표시됩니다.

Blog: 블로그 모듈은 외부 계정 연결이 불필요. 정상 경로로 진행.
