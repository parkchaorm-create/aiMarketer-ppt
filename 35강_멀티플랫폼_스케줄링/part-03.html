18강에서 3축(Social Planner · Blog · MCP)을 다뤘습니다. 35강은 거기에 하나를 더합니다 — Advanced CSV 업로드입니다.

18강에서는 3채널 × 1건 = 총 3개 예약 슬롯이었어요. 오늘은 7일 × 3채널 = 최대 21개 슬롯입니다. UI로 21개를 단건 입력하면 한 회차에 끝나지 않아요. 그래서 CSV 업로드 한 번에 5~15개를 한꺼번에 예약합니다.

공식 CSV는 두 종류입니다[^4]:

Basic CSV: Date · Content · OG Meta URL · Image/Video Links · GIF (5개 칼럼)

Advanced CSV: 플랫폼별 포스트 타입 · 워터마크 · 태그 · 카테고리 · 팔로업 댓글 · GBP 설정 · YouTube 메타 · TikTok · LinkedIn PDF · Pinterest 보드 (20+ 칼럼)

오늘 실습 B는 Basic CSV로 시작합니다. Advanced CSV는 템플릿만 훑어보고, 팀별로 필요할 때 나중에 확장하시면 됩니다.

"포트폴리오 URL을 게시물마다 CTA로 넣는다"고만 하면 몇 명이 어느 채널에서 들어왔는지 구분이 안 됩니다. 그래서 URL에 UTM 파라미터(UTM Parameter · 유입 경로 추적용 쿼리스트링)를 붙입니다.

https://[포트폴리오URL]/?utm_source=instagram&utm_medium=social&utm_campaign=launch-week1

CONCEPT · 카드 2 · UTM 3요소
03 / 07

source · medium · campaign

  • 01 🔍 utm_source · 어디에서 →
    어디에서 왔는가(플랫폼)입니다. `instagram` · `facebook` · `blog` 같은 채널 이름이 값으로 들어갑니다.
  • 02 📡 utm_medium · 어떤 방식 →
    어떤 매체 종류인가입니다. `social`(SNS) · `organic`(블로그·검색) · `email` 같은 큰 카테고리가 값입니다.
  • 03 🏷️ utm_campaign · 묶음 이름 →
    어떤 캠페인 묶음인가입니다. 팀당 한 가지로 고정: `purut-launch` · `slowtempo-launch` · `sonkyul-launch` · `linkedmind-launch`.
  • 04 ⚠️ 규칙 · 소문자·영문·하이픈 →
    값은 소문자·영문·하이픈만 사용합니다. 한글·공백 금지. `?`로 시작 · `&`로 연결. GA4·GHL Sites Analytics가 자동 집계합니다.

신규 최대 7 · 재활용 14

  • 01 🖼️ 7강 · 카드뉴스 원본 재활용 →
    팀원별 1세트 · 팀 전체 3~5세트의 카드뉴스가 인스타그램 게시물 원본 + 캡션으로 그대로 재활용됩니다.
  • 02 📆 8강 · 글 5건 + 캘린더 →
    글 5건 + 카드뉴스 2세트 + 캘린더 1장이 페이스북 글 원본과 채널별 변형 기초로 쓰입니다.
  • 03 📂 9강 · 포트폴리오 PDF →
    포트폴리오 패키지 PDF/웹을 블로그 롱폼 재가공과 핵심 발췌 재료로 사용합니다.
  • 04 ✨ 신규 ≤ 7건 · 상한 →
    포트폴리오 URL 리마인드·팀 소개·롱폼·하이라이트·서비스 3종·팀 크레딧·Behind the Scenes. 7건이 상한입니다.

7일 × 3채널 · 권장 템플릿

  • 01 📆 월·목 · 중량 콘텐츠 →
    월요일 · 목요일에 중량 콘텐츠 집중 배치. 7강 카드뉴스 재활용 + 신규 서비스 3종 요약을 메인 슬롯에 올립니다.
  • 02 📝 화·금·일 · 블로그 롱폼 →
    블로그는 주 3회(화·금·일) 롱폼만 배치합니다. 9강 PDF 재가공 · 첫 1주일 롱폼 · BTS 숏폼이 기본 구성입니다.
  • 03 🌿 토·일 · 가벼움 + 스토리 →
    토·일은 가볍게 운영합니다. 인스타 스토리형 URL 리마인드와 팀 크레딧 캐러셀처럼 부담 없는 콘텐츠로 주말을 덮습니다.
  • 04 🎯 휴지일 0 · URL 3건 이상 →
    휴지일은 0일로 유지하고, 포트폴리오 URL은 최소 3건 게시물에 UTM 포함으로 들어가면 측정이 유의미해집니다.

GHL Social Planner UI / CSV 업로드 = 예약 슬롯 확정 무대

→ 명령어가 재료를 써 주고, 수강생이 GHL에 복사·붙여넣기 또는 업로드로 확정

"슬래시 명령어가 버튼 한 번으로 Social Planner에 직접 꽂히나요?" 라는 질문이 종종 나옵니다. 공식 문서는 MCP의 "Create Post"가 Social Planner 예약 슬롯에 직접 꽂히는지 세부 동작을 명시하지 않았습니다[^5]. 그래서 오늘도 하이브리드 원칙을 유지합니다 — 초안은 AI에게, 확정은 사람이.

지도를 다 그렸습니다. 이제 실습으로 넘어갑니다. 실습 A에서 7일 캘린더 초안을 만들고, UI로 3~5건을 단건 예약해 볼게요. 실습 B에서 CSV 업로드 1건과 신규 슬래시 명령어 1건을 추가합니다.
