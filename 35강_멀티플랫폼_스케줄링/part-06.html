CSV 업로드 흐름

  • 01 📂 5단계 · Bulk Schedule →
    명령어 호출 → CSV 작성 → Bulk Upload(대량 업로드 · CSV 한 장으로 여러 게시물 한꺼번에 예약) → Preview → Confirm. 다섯 클릭으로 10건이 한 번에 들어갑니다.
  • 02 📊 Basic CSV · 5칼럼 →
    Date · Content · OG Meta URL · Image/Video Links · GIF 다섯 칼럼이 기본형(Basic CSV · 최소 양식)입니다. 오늘 실습은 이 5칼럼만 채웁니다.
  • 03 ⚙️ 행 5~10 · 다채널 분기 →
    한 CSV에 인스타·페북·블로그 행을 섞어 5~10개 작성합니다. 행마다 채널 라우팅(행별로 다른 플랫폼으로 분배)되어 각 채널 슬롯에 자동 분배됩니다.

02 / 07

5칼럼 · 행 작성법

  • 01 📅 Date · 분 단위 →
    YYYY-MM-DD HH:MM 형식으로 분 단위까지 적습니다. 예약 시점은 수업 종료 이후 7일 범위에 분산하고, 같은 시각에 두 행이 겹치지 않게 합니다.
  • 02 ✍️ Content · 캡션 본문 →
    게시물 캡션을 큰따옴표로 감쌉니다. 줄바꿈은 그대로 살릴 수 있고, 채널마다 길이를 달리해 행을 따로 만드시면 됩니다.
  • 03 🔗 OG Meta URL · UTM →
    포트폴리오 URL에 UTM 3요소(source·medium·campaign)를 붙여 넣습니다. 채널마다 utm_source 값만 바꾸면 한 캠페인을 채널별로 추적할 수 있습니다.
  • 04 🖼️ Image/Video · GIF →
    이미지·영상 링크와 GIF 칼럼은 비워도 업로드는 됩니다. 7강 카드뉴스 이미지 URL을 붙여 두면 한 행에 비주얼까지 한꺼번에 올라갑니다.

04 / 07

Preview · Confirm 분기

  • 01 👀 Preview · 행 단위 검수 →
    Preview(미리보기 · CSV 업로드 직후 행별로 표시되는 검수 화면)에서 채널·시간·캡션·UTM이 의도대로 들어갔는지 행 단위로 확인합니다.
  • 02 ✅ Confirm · 일괄 예약 확정 →
    Confirm 버튼으로 모든 행이 한 번에 Schedule for Later 상태가 됩니다. 이 시점부터 Calendar 화면에 점으로 표시되어 36강 시연 자료가 됩니다.
  • 03 ⚠️ 실패 행만 분리 재업로드 →
    CSV 포맷 오류가 있으면 GHL이 실패 행만 리턴해 줍니다. 전량을 다시 올리지 마시고, 해당 행만 시트에서 고쳐 그 행만 따로 재업로드하시면 됩니다.
  • 04 🔁 Recurring · 선택 1건 →
    여유 팀은 Recurring(반복 게시 · 같은 글을 일·주·월 단위로 자동 재게시) 1건을 매주 금요일 10시 4주짜리로 추가합니다. 종료는 수기로 끄셔야 합니다.

CSV 흐름 · 3줄

  • 01 📂 Bulk 업로드 1건 확정 →
    Basic CSV 5칼럼에 행 5~10개를 채워 한 번의 Bulk Upload(대량 업로드)로 일괄 예약했습니다. UI 단건 예약 대비 시간이 크게 줄었습니다.
  • 02 ⌨️ 명령어 2건 · 재호출+신규 →
    24강 /schedule-post 인자 치환 재호출 1회 + 신규 /weekly-launch-plan.md 파일 1개 추가. 두 명령어가 36강 이후에도 캠페인마다 재사용됩니다.
  • 03 ✅ 다채널 분기 · UTM 일관 →
    한 CSV 안에 인스타·페북·블로그 행이 섞여 들어갔고, utm_campaign 값이 팀 전체에서 한 가지로 일관되었습니다. 36강 피칭 증빙 준비 완료입니다.

Q2. UTM 파라미터가 캡션에 보이면 지저분하지 않나요? A. 인스타그램·페이스북은 링크 미리보기에 URL이 정리되어 보여서 캡션에는 원본 URL만 노출되고 UTM은 클릭 시 전달됩니다. 블로그는 하이퍼링크로 감싸면 UTM이 캡션에 드러나지 않습니다.
