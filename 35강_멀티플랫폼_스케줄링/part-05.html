UI 단건 예약 흐름

  • 01 📅 5단계 · UI 단건 예약 →
    Marketing → Social Planner → Calendar → New Post → Customize for Each Post → Schedule for Later. 다섯 클릭이 오늘의 핵심 동선입니다.
  • 02 🎯 3~5건 · 분산 시간대 →
    기존 자산 1~2건 + 신규 초안 2~3건 합쳐 3~5건을 수업 종료 이후 7일 범위 내 분산해 Schedule for Later로 확정합니다.

    최소 3건 게시물에 포트폴리오 URL이 UTM 파라미터(utm_source·medium·campaign · 유입 경로 추적용 쿼리스트링)와 함께 들어가야 합니다.

Customize for Each Post

  • 01 📷 인스타그램 · 짧게 →
    짧은 캡션 + 해시태그 3~5개 + 카드뉴스 이미지. 7강 카드뉴스 자산을 그대로 재활용하고 캡션만 새로 다듬습니다.
  • 02 📘 페이스북 · 중간 길이 →
    맥락 설명을 포함한 중간 길이 텍스트와 포트폴리오 URL 링크 프리뷰. 8강 글 원본을 가져와 첫 문단만 살짝 손보면 됩니다.
  • 03 📝 블로그 · 롱폼 500자 →
    제목 + 부제 + 본문 500자 이상 + SEO 키워드. 9강 포트폴리오 패키지 PDF에서 핵심 발췌해 롱폼으로 재구성합니다.
  • 04 ⚠️ 동일 카피 복붙 금지 →
    18강 학습대로 같은 글을 3채널 동일 복사 붙여넣기는 알고리즘 노출에도 불리합니다. Customize 토글을 켜고 채널별로 손봐 주세요.

시간 · 4가지 체크

  • 01 ⏰ Best Time · 청중 데이터 →
    Best Time to Post(소셜 플래너 내장 기능 · 청중 참여 데이터 기반 최적 시간 추천)가 있지만, 데이터가 쌓이지 않은 팀은 권장 시간대(출근·퇴근 직후)를 강제 적용해도 됩니다.
  • 02 ✅ 체크 1 · URL+UTM →
    예약 게시물에 포트폴리오 URL이 UTM 파라미터 포함으로 들어갔는지 매번 확인합니다. Part 04에서 시트에 미리 붙여 둔 URL을 복사해 쓰세요.
  • 03 ✅ 체크 2 · 채널 맞춤 카피 →
    Customize for Each Post 토글이 켜져 있고, 인스타·페북·블로그 카피가 서로 다른 길이·톤으로 작성되어 있는지 확인합니다.
  • 04 ✅ 체크 3 · 시간 분산 7일 →
    3~5건이 같은 날에 몰리지 않고 7일 범위 내 분산되었는지 확인합니다. 월·목 중량, 토·일 가벼움 원칙을 유지합니다.

막혀도 통과 · 3 시나리오

  • 01 🛟 계정 미연동 → Blog 1건 →
    34강에서 SNS 계정 연동이 안 된 팀은 Blog 단건 예약 1건만 수행해도 실습 A 통과로 인정됩니다. SNS는 36강 이전까지 별도 시간에 보충하시면 됩니다.
  • 02 🛟 저장만 됨 → Save Later →
    계정 연동은 됐지만 게시물이 저장만 되고 예약이 안 되는 팀은 Save for Later 상태로 두세요. 수업 후 담당 강사가 개별 순회 지원합니다.
  • 03 🛟 URL 미라이브 → 임시 →
    포트폴리오 URL이 라이브 전이면 Stitch 미배포 프리뷰 또는 팀 구글 드라이브 PDF 공유 URL을 임시로 사용해 UTM 파라미터만 먼저 세팅합니다.
