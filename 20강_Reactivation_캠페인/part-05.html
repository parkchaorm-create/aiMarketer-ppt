필터 1개, 진짜로 저장할 수 있을까요?

  • 01 👀 시연 먼저 · 강사 전면 화면 →
    Advanced Filters → 조건 추가 → Smart List 저장까지 강사가 전면 화면에서 한 사이클 시연합니다.
  • 02 🖐️ 수강생 · 각자 자리에서 →
    시연 후 수강생 각자 자리에서 본인 가상 클라이언트용 Smart List 1개를 저장합니다. 실습 중 화면 공유 환영이에요.
  • 03 📤 슬랙 · [20강][A/B/C/D] →
    완성되면 `#수업실습` 채널에 `[20강][A/B/C/D] 필터규칙 · 조건요약` 형식으로 업로드해 서로 참고합니다.

02 / 07

오늘 실습의 네 가지 약속

  • 01 ⏱️ 1) 시간 · 시연 짧게 · 개별 김 →
    시연은 6단계 핵심 지점만 짧게. 대부분의 시간은 수강생이 각자 자리에서 본인 클라이언트로 만드는 데 씁니다.
  • 02 🔓 2) 화면 공유 · 환영 →
    실습 중 자유롭게 화면을 공유해 주세요. 서로의 조건 조합이 참고가 되고, 강사·운영진이 빠르게 돕기도 쉬워요.
  • 03 🧑‍🤝‍🧑 3) 동료 참고 · 베끼기 OK →
    옆자리 수강생의 조건을 참고해서 시작해도 됩니다. 운영 중 다듬으면 되고, 첫 시도는 빠르게 돌려 보는 것이 더 중요해요.
  • 04 🆘 4) 막힘 · 손들기 · QnA 채널 →
    막히는 분은 `#강사-QnA` 채널에 증상 한 줄 + 스크린샷을 올리거나, 직접 손을 들어 강사·운영진을 호출해 주세요.

03 / 07

슬랙 · 한 줄 포맷 · 고정

  • 01 📝 1) 채널 · #수업실습 고정 →
    산출물 업로드는 `#수업실습` 채널 전용. `#질문`·`#자유`와 분리되어 있어 강사·운영진이 놓치지 않고 피드백할 수 있어요.
  • 02 🏷️ 2) 접두 태그 · [20강][A/B/C/D] →
    게시글 맨 앞에 `[20강][A/B/C/D]` 접두 태그를 붙여 주세요. 검색과 회차별 정렬이 쉬워집니다.
  • 03 📋 3) 조건 요약 · 4줄 이내 →
    Smart List 이름 + 기준일 + 태그 + 제외 조건 — 4줄 이내로 요약해 올립니다. 스크린샷은 선택사항이에요.

04 / 07

합격 체크 · 세 가지만

  • 01 ✅ 1) Smart List 저장 · 이름 있음 →
    Contacts 메뉴 좌측 Smart Lists 탭에 본인 이름의 Smart List가 하나 떠 있으면 OK. 수치는 나중에 다듬어도 됩니다.
  • 02 ✅ 2) 제외 조건 · 최소 2개 →
    unsubscribed와 converted · 이 두 제외 조건은 필수예요. 나중에 재활성화 발송 시 수신거부자 보호의 1차 선입니다.
  • 03 ✅ 3) 슬랙 · 4줄 요약 업로드 →
    `#수업실습` 채널에 `[20강][A/B/C/D]` 접두 + 4줄 요약이 올라가면 실습 A 완료. 강사·운영진이 확인 후 피드백합니다.

05 / 07

실습 중 · 법 준수 2가지

  • 01 ✅ 동의 · 제외 조건에 반영 →
    Smart List에 `NOT Tag: unsubscribed`가 들어가 있어야 해요. 수신거부자는 발송 대상에서 즉시 제외됩니다.
  • 02 📤 수신거부 · Part 05 메시지에서 →
    메시지 본문에는 수신거부 링크를 반드시 넣습니다. 필터 단계가 아닌 메시지 단계에서 처리 — Part 05(실습 B)에서 실습합니다.
  • 03 🛑 실제 연락처 · 등록 금지 →
    오늘 실습에 본인 실제 고객의 전화·이메일 등록은 금지예요. 테스트는 가상 형식(`test-닉네임@example.com`)으로만.

Part 05 · 세 줄 정리

  • 01 🎬 시연 · 개별 · 업로드 →
    강사 시연 짧게 → 수강생 개별 작업 길게 → 슬랙 업로드까지. 이 세 박자가 오늘 실습 A의 리듬입니다.
  • 02 📋 4줄 요약 · 고정 포맷 →
    이름·기준일·태그·제외 — 4줄만 `#수업실습`에 `[20강][A/B/C/D]` 접두와 함께 올려 주세요.
  • 03 ⚖️ 법 2축 · 제외 + 금지 →
    수신거부 태그는 필터 제외 조건으로, 수신거부 링크는 메시지 본문으로. 실제 고객 연락처 등록은 금지입니다.
