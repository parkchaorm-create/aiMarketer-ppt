💬 그대로 읽으세요
1. 가상 클라이언트 확인 — 1강에서 선택한 A/B/C/D 중 하나. 업종에 따라 기본 기준일(60일/90일) 결정. 2. GHL Contacts 탭 진입 — 좌측 메뉴 Contacts → 상단 Advanced Filters 버튼 클릭. 3. 필터 조건 구성 — `Last Activity`, `Tag`, 제외 조건(unsubscribed·converted 등)을 AND로 조합. 4. 필터 결과 미리보기 — 조건 저장 전에 실제 매칭되는 연락처 수 확인 (16~17강에서 테스트 등록한 리드가 보일 수 있음). 5. Smart List로 저장 — "Save as Smart List" 버튼 → 이름 입력. 권장 이름: `재활성화-대기-60일` 또는 `푸르트_방치리드_90일`. 6. 공유 링크 또는 스크린샷 — 조건 요약을 슬랙 `#수업실습` 채널에 `[20강][A/B/C/D] 필터규칙` 형식으로 업로드.

예시 1 · 푸르트(소매) — 60일 기준

Last Activity > 60 days ago AND Tag contains "need-nurturing" OR Tag contains "high-intent" AND Tag not contains "unsubscribed" AND Tag not contains "converted"

확인 → 진입 → 조건 → 미리보기 → 저장 → 업로드

  • 01 👤 1) 클라이언트 확인 · A/B/C/D →
    1강에서 고른 가상 클라이언트 A 푸르트·B 슬로우템포·C 손결·D 링크드마인드 중 하나. 업종에 따라 기본 기준일이 정해집니다.
  • 02 🚪 2) GHL Contacts · Advanced Filters →
    좌측 메뉴 Contacts 탭 → 상단 Advanced Filters 버튼을 클릭해 조건 구성 창을 엽니다.
  • 03 🧩 3) 조건 구성 · AND 조합 →
    Last Activity · Tag · 제외 조건(unsubscribed·converted)을 AND로 조합합니다. Part 03에서 본 3축 그대로.
  • 04 👀 4) 결과 미리보기 · 저장 전 →
    조건 저장 전에 실제 매칭되는 연락처 수를 먼저 확인합니다. 16~17강에서 등록한 테스트 리드가 보일 수 있어요.
  • 05 💾 5) Save as Smart List · 이름 →
    "Save as Smart List" 버튼 → 이름 입력. 권장 이름: `재활성화-대기-60일` 또는 `푸르트_방치리드_90일`.

조건 초안 · 복사용 프롬프트

  • 01 📋 용도 · Smart List 초안 받기 →
    Claude Code(클로드 코드 · 터미널 기반 AI 코딩 에이전트)에게 Smart List 조건 초안을 받고 싶을 때 쓰는 프롬프트예요.
  • 02 🧱 입력 4요소 · 치환만 하면 OK →
    클라이언트·방치 기준일·활용할 태그·제외 조건 — 대괄호 4개만 본인 가상 클라이언트에 맞춰 교체하세요.
  • 03 🎯 출력 · AND/OR 논리 목록 →
    출력 형식을 "Advanced Filters에 그대로 옮길 수 있는 AND/OR 논리 조건 목록 + 한국어 주석 1줄"로 지정하면 실무 바로 사용 가능해요.
  • 04 💡 실제 저장 · GHL UI로 →
    클로드 응답은 참고용 초안. GHL Advanced Filters는 클릭·드롭다운 방식이라 UI에서 직접 조합하는 것이 더 정확해요.

자주 나오는 4가지 · 1차 조치

  • 01 🔍 Last Activity · 필드가 안 보임 →
    좌측 필터 그룹에서 Activity 섹션을 스크롤해 펼쳐 주세요. 하위 그룹에 숨겨져 있는 경우가 많습니다.
  • 02 🕳️ 매칭 결과 0명 · 정상일 수도 →
    16·17강 테스트 리드가 아직 60일 방치 조건을 못 만족해서 그래요. `> 1 days`로 임시 축소해 로직 동작만 확인하세요.
  • 03 ⚪ Save 버튼 회색 · 조건 비활성 →
    최소 한 개 조건이 활성화되지 않은 상태입니다. 조건 줄 왼쪽 체크 표시가 들어 있는지 확인해 주세요.
  • 04 📂 저장 후 안 보임 · Smart Lists 탭 →
    Contacts 메뉴 좌측의 Smart Lists 탭으로 전환해야 저장된 목록이 보입니다. 기본 탭에서는 표시되지 않아요.

Part 04 · 세 줄 정리

  • 01 🗺️ 6단계 · 저장·명명까지 합격 →
    클라이언트 확인 → Advanced Filters → AND 조합 → 미리보기 → Save as Smart List → 슬랙 업로드. 수치 조정은 운영하면서.

    푸르트 60일 4줄 또는 손결 90일 + 열람 4~5줄. 가상 클라이언트에 맞춰 하나를 기본값으로 시작합니다.

    Claude Code 프롬프트로 조건 초안을 빠르게 받되, 실제 저장은 GHL UI 클릭·드롭다운에서 진행합니다.

💬 그대로 읽으세요
1. 강사가 전면 화면에서 6단계 중 핵심 지점을 시연합니다 (Advanced Filters → 조건 추가 → Smart List 저장). 2. 시연 후 수강생은 각자 자리에서 본인 가상 클라이언트용 Smart List 1개를 저장합니다. 3. 완성된 수강생은 슬랙 `#수업실습` 채널에 `[20강][A/B/C/D] 필터규칙 · 조건요약` 형식으로 업로드. 실습 중에도 진행 화면을 자유롭게 공유해 주시면 서로 참고가 됩니다. 4. 막히는 분은 `#강사-QnA` 채널 또는 손 들기로 강사·운영진 호출.

우리는 "방치 + 태그 + 제외"를 Smart List 1개로 저장했어요.

기본값은 60일 또는 90일. 업종에 따라 조정합니다.
