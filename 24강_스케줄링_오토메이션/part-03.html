CONCEPT · 3경로 지도
03 / 07

병합 경로 · 세 가지 비교

  • 01 1️⃣ 단일 분기 · If/Else 한 판 →
    한 워크플로우 안에 If/Else(조건 분기) 노드로 스케줄링·Reactivation을 모두 처리. 시야는 한 화면, 대신 디버깅이 어렵습니다.
  • 02 2️⃣ 태그 체이닝 · A→태그→B →
    워크플로우 A가 끝나며 태그 추가 → 태그를 Trigger(트리거 · 자동화 시작 조건)로 워크플로우 B가 자동 시작. 오늘의 주경로입니다.
  • 03 3️⃣ 명령어 오케스트레이션 →
    Slash Command가 각 워크플로우의 트리거 조건·입력 데이터를 구성. 실습 B에서 `.claude/commands/` 폴더에 3종 저장해 봅니다.

CONCEPT · 슬래시 명령어
04 / 07

`.claude/commands/` 한 폴더

  • 01 📄 파일 이름 · 그대로 명령어 →
    `schedule-post.md` 파일을 저장하면 Claude Code 터미널에서 `/schedule-post`로 호출됩니다. 파일 본문이 프롬프트로 전달됩니다.
  • 02 🔣 $ARGUMENTS · 인자 치환 →
    `/schedule-post 푸르트 5월` 입력 시 파일 안의 $ARGUMENTS가 "푸르트 5월"로 치환됩니다. 같은 프롬프트를 여러 재료로 재활용.
  • 03 ⚠️ 프롬프트 저장고 · 자동 실행 X →
    Slash Command는 프롬프트 저장고입니다. 명령어 하나로 GHL 워크플로우가 자동 실행되지는 않습니다. 저장·확정은 GHL UI에서.
