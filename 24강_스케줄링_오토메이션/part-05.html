1. 폴더 확인 — 팀 프로젝트 루트에 `.claude/commands/` 폴더가 있는지 확인. 없으면 1줄로 생성. 2. 명령어 파일 3개 저장 — `schedule-post.md` · `reactivate-lead.md` · `merge-automation.md` 각각 생성. 3. 명령어 호출 테스트 — Claude Code 터미널에서 `/schedule-post`·`/reactivate-lead`·`/merge-automation` 순서대로 호출하여 응답 확인. 4. 팀 공유 — 파일 3개를 팀 Git 저장소(또는 팀 드라이브)에 커밋·공유. 이후 팀원 누구나 같은 명령어 사용 가능.

역할: 18강 자산(3채널 예약 콘텐츠 초안 프롬프트)을 호출하여 다음 주 예약 콘텐츠 초안 생성

1. `.claude/commands/` 폴더 안의 마크다운 파일 3개 — `schedule-post.md`·`reactivate-lead.md`·`merge-automation.md` 2. 각 명령어 호출 결과 스크린샷 3장 — Claude Code 터미널에서 3개 명령어를 각각 호출하여 응답이 나온 화면.

슬랙 `#수업실습`에 `[24강][A/B/C/D] 슬래시명령어카탈로그·3종·호출결과` 형식으로 업로드. 팀 전원이 같은 `.claude/commands/` 폴더를 공유한다면 팀 대표 1명이 스크린샷 3장을 올려 주시면 됩니다.

1. 강사가 `schedule-post.md` 파일 생성 → Claude Code 터미널에서 `/schedule-post 푸르트 · 5월 월간 꽃 구독 오픈` 호출 → 응답 확인까지 한 사이클 시연. 2. 수강생은 시연한 `/schedule-post`를 먼저 저장하고 1회 호출. 정상 동작 확인 후 `/reactivate-lead`·`/merge-automation` 같은 패턴으로 반복. 3. 3개 파일 모두 저장되면 팀 Git 저장소에 커밋(또는 드라이브 공유). 4. 스크린샷 3장 슬랙 업로드.

Claude Code에 GHL MCP(Model Context Protocol) 서버가 연결되어 있으면[^6], `/mcp__ghl__create_workflow` 같은 형식의 MCP 도구 호출 슬래시 명령어가 자동으로 노출됩니다. 오늘은 직접 실행하지 않지만, 다음 단계에 관심 있는 팀을 위해 경로만 기록합니다.

// 공식 엔드포인트 (2026-04 기준) GHL MCP Server URL: https://services.leadconnectorhq.com/mcp/ 인증: GHL Private Integration Token (Settings → Private Integrations에서 발급)

`.claude/commands/` 폴더에 마크다운 3개 저장 = 팀 공유 슬래시 명령어 카탈로그 완성

3종 역할 분담: `/schedule-post` 콘텐츠 초안 · `/reactivate-lead` 재활성화 초안 · `/merge-automation` 병합 체크리스트

슬래시 명령어는 프롬프트 저장고. 워크플로우 저장·확정은 GHL UI에서.

MCP 도구 직접 호출은 심화 · 테스트 Sub-account에서만.

두 실습이 끝났으니, 오늘 만든 것을 정리하고 옆자리 팀원과 짧게 공유해 보는 시간을 갖겠습니다.
