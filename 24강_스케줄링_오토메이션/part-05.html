    터미널에서 3개 명령어 순서대로 호출하여 응답 확인. 팀 Git에 커밋하면 팀원 누구나 같은 명령어 사용 가능.

매번 · 손으로 · 다시 그리실 건가요?

  • 01 ✍️ 다음 주에도 · 같은 프롬프트? →
    팀이 다음 주에도, 다다음 달에도 새 클라이언트에 같은 프롬프트를 타이핑해야 한다면 시간 낭비입니다.
  • 02 🗂️ 저장 · 한 번 · 호출 · 무한 →
    마크다운 한 장 저장 = 팀 공유 슬래시 명령어 한 개. 다음부터는 `/명령어` 입력만으로 같은 프롬프트를 꺼냅니다.
  • 03 🧰 팀 자산 · 처음으로 · 공유 →
    `.claude/commands/` 폴더가 팀 자산의 출발점. 오늘이 팀 차원에서 재사용 가능한 프롬프트 저장고를 만드는 첫날입니다.

  • 01 📢 /schedule-post · 3채널 초안 →

    두 워크플로우 태그 기반 병합 절차를 체크리스트 + UI 확정 가이드 형식으로 출력. 중복 방지 3조건 · 법 5체크 자동 포함.

04 / 07

자주 · 막히는 · 네 지점

  • 01 🔄 명령어 미인식 · Claude 재시작 →
    파일을 저장했는데 `/schedule-post`가 인식 안 될 때. Claude Code 재시작 후 `/help`로 명령어 목록을 확인하면 대부분 해결됩니다.
  • 02 📐 Frontmatter 구분자 · `---` 누락 →
    파일 저장됐는데 빈 응답이 나오면 YAML Frontmatter의 `---` 구분자가 맨 위·메타데이터 아래 각각 있는지 먼저 확인하세요.
  • 03 🔤 $ARGUMENTS 치환 실패 →
    명령어 뒤에 인자를 붙이지 않았거나 $ARGUMENTS가 코드 블록 안에 갇혀 있으면 치환이 안 됩니다. 블록 밖에 배치하세요.
  • 04 🏠 홈 디렉터리로 저장 · 이동 →
    파일이 `~/.claude/commands/`(홈)에 저장되면 팀원 공유가 안 됩니다. 팀 프로젝트 루트 `./.claude/commands/`로 이동하세요.
