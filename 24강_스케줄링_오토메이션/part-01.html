💬 그대로 읽으세요
지난 23강에서 팀별로 파이프라인 아키텍처 문서 + 주요 노드 3개 이상을 빌드해 두셨습니다. 그 블록들이 지금은 각자 따로 서 있습니다. 오늘은 그 블록들 사이를 다리로 잇는 날입니다. 팀이 지난 회차까지 만든 18강 예약 발행 엔진과 20강 Reactivation 엔진을, 23강 파이프라인 위에 하나의 흐름으로 병합합니다.

1. 🔗 통합 오토메이션 1개 — 23강 팀 파이프라인에 18강(스케줄링)·20강(리드 활성화) 자산을 태그 기반 체이닝(Tag-based Chaining · 태그로 이어 붙이기)으로 연결합니다. 팀당 워크플로우 2~3개가 서로 물리는 구조 1개 완성. 2. ⌨️ 클로드 코드 슬래시 명령어 카탈로그 3종 — `.claude/commands/` 폴더에 마크다운 파일 3개를 저장합니다. 각 파일은 사전 빌드된 프롬프트이며, Claude Code 터미널에서 `/schedule-post`·`/reactivate-lead`·`/merge-automation`으로 호출됩니다.

자동화 5개, 매번 따로 켜고 계신가요?

  • 01 🧱 블록은 있는데 · 사이가 비어요 →
    23강에서 팀별 Pipeline(파이프라인 · 고객 여정 단계) + 주요 노드 3개 이상을 빌드했지만, 블록끼리는 아직 따로 서 있습니다.
  • 02 💥 사이가 비면 · 실무 사고 3가지 →
    같은 날 3개 워크플로우가 동시 발송 · Reactivation 응답자에게 예약 피드가 그대로 흐름 · VIP 후보 폼 제출 누락. 번거로움이 아니라 사고입니다.
  • 03 🌉 다리를 놓는 날 · 오늘 한 번 →
    차이는 워크플로우 개수가 아니라, 워크플로우끼리 말을 주고받는지에서 갈립니다. 오늘은 말길을 여는 날입니다.

팀 0단계 · 4가지 확인

  • 01 📄 파이프라인 아키텍처 문서 →
    23강에서 만든 팀 파이프라인 아키텍처 문서(구글 드라이브·노션·Figma 중 하나)가 팀 모니터에 열려 있어야 합니다.
  • 02 ⚙️ GHL 워크플로우 3개 이상 →
    GHL(GoHighLevel · 올인원 마케팅 자동화 플랫폼) 워크플로우 3개 이상이 Draft 상태라도 존재해야 합니다. 최소 1개는 23강에서 빌드.
  • 03 👤 가상 클라이언트 1종 고정 →
    A 푸르트 · B 슬로우템포 · C 손결 · D 링크드마인드 중 팀이 선택한 클라이언트 1종이 오늘 내내 기준이 됩니다.
  • 04 📁 팀 `.claude/commands/` 존재 →
    10강 결과물인 `.claude/` 폴더 안에 `commands/` 하위 폴더가 있어야 실습 B가 바로 이어집니다.

오늘 만들지 않는 것 · 4줄

  • 01 🚫 새 워크플로우 · 처음부터 X →
    이미 만든 것을 연결만 합니다. 새로 만드는 유혹을 먼저 끊어야 2시간 안에 다리가 완성됩니다.
  • 02 🚫 실 고객 대상 발송 · 금지 →
    테스트 Lead(리드 · 잠재 고객 데이터)와 가상 주소만 사용합니다. 실 발송은 오늘의 안전 경계 밖입니다.
  • 03 🚫 자동화 10개 한 번에 · 금지 →
    오늘은 연결 지점 2~3개가 상한. 세 개 대충 잇는 것보다 한 개를 확실히 잇는 것이 훨씬 중요합니다.
  • 04 🚫 Snapshot 가져오기 · 25강 →
    GHL Snapshot(스냅샷 · 템플릿 복제 도구) 가져오기는 25강·포스트 회차 주제. 오늘은 팀 자체 자산만 만집니다.
