세 가지 · 팀 사전 합의

  • 01 📍 연결 지점 · 3개 지목 →
    파이프라인 그림에서 "여기서 여기로 이어져야 한다" 선을 빨간펜으로 3개만 표시합니다. 오늘의 상한은 3개입니다.
  • 02 🏷️ 태그 표준 · 3개 합의 →
    팀이 쓸 연결 태그 이름 3개 고정. 띄어쓰기 대신 하이픈(-) 권장. 예: `이번주-수신완료`·`재활성화-T1-발송됨`·`vip-제안-발송됨`.
  • 03 🗺️ 6단계 사이클 · 지도 →
    0단계 팀 산출물 열기부터 6단계 테스트 리드 체인 확인까지 큰 지도를 먼저 본 뒤 노션 튜토리얼로 세부 단계 진행.

02 / 07

세 연결 · 완벽하게 이어야 하나요?

  • 01 🛟 실패 안전망 · 한 연결이면 OK →
    수업 시간 내 목표는 "워크플로우 A 마지막 액션이 B 트리거로 도달하는 한 연결" 한 개 확인입니다. 완벽하게가 아니라 일단 도달.
  • 02 🔄 두 번째·세 번째 · 25강에서 →
    두 번째·세 번째 연결은 25강 라이브 디버깅에서 보정합니다. 오늘은 첫 연결에 집중, 다음 회차 부하 테스트로 넘깁니다.
  • 03 🧪 결과 다름 · 정상입니다 →
    Stochastic Output(확률적 결과물 · AI 생성물은 매번 조금씩 다를 수 있음). 결과가 기대와 다르게 나와도 정상입니다.

03 / 07

팀 단위 · 6단계 조감

  • 01 0️⃣1️⃣ 팀 산출물 열기 + 지목 →
    0단계: 파이프라인 문서·워크플로우 목록·클라이언트 카드 열기. 1단계: 연결 지점 3개 빨간펜 표시. 순서가 중요합니다.
  • 02 2️⃣3️⃣ 태그 표준 + 액션 추가 →
    2단계: 태그 이름 3개 합의. 3단계: 18강 예약 워크플로우 말미에 Add Contact Tag 액션 추가 후 Draft 저장.
  • 03 4️⃣5️⃣ 트리거 갱신 + 체인 확인 →
    4단계: 20강 Reactivation 워크플로우 트리거에 Tag Contains 조건 추가 + 중복 방지 3조건. 5단계: 테스트 리드 1명으로 체인 1회 확인.

04 / 07

설정 시점에 · 이미 심기

  • 01 🚫 ① unsubscribed · 수신거부 →
    모든 워크플로우 트리거에 `Tag NOT contains "unsubscribed"`. 수신거부자에게는 어떤 경우에도 메시지가 가지 않는 1차 벽.
  • 02 ✅ ② converted · 전환 완료 →
    `Tag NOT contains "converted"`. 이미 전환된 리드에게 재활성화·DM 캠페인이 돌아가지 않도록 2차 벽.
  • 03 📅 ③ 지난 7일 · 수신 이력 →
    `Tag NOT contains "이번주-수신완료-지난7일"` + Time-based Tag Removal(시간 기반 태그 제거 · 7일 뒤 자동 제거) 액션 조합.
