OVERVIEW
📝 강사 노트
💬 그대로 읽으세요
실습 A는 팀 단위로 6단계 진행합니다. 세부 단계는 노션 튜토리얼(`실습_튜토리얼/실습A_팀워크플로우_병합.md`)에 화면 캡처와 함께 정리됩니다. 여기서는 큰 지도만 보고 갑니다.

1. 0단계 · 팀 산출물 열기 — 23강 파이프라인 아키텍처 문서 + GHL 워크플로우 목록 + 가상 클라이언트 카드. 팀장이 팀 모니터 한쪽에 띄워 둡니다. 2. 연결 지점 3개 지목 — 파이프라인 그림에서 "여기서 여기로 이어져야 한다" 선을 빨간펜으로 3개만 표시합니다. 많이 그리지 마시고 오늘의 상한은 3개. 3. 태그 표준 합의 — 팀이 사용할 연결 태그 이름 표준을 3개 정합니다. 예: `이번주-수신완료`·`재활성화-T1-발송됨`·`vip-제안-발송됨`. 띄어쓰기 대신 하이픈(-) 권장, 팀 전체가 같은 이름 사용. 4. 워크플로우 A 수정 — 마지막 액션에 Add Contact Tag 삽입 — 18강 예약 발행 워크플로우 말미에 `Add Contact Tag: 이번주-수신완료` 액션을 추가하고 Draft 저장[^8]. 5. 워크플로우 B 트리거 조건 갱신 — 20강 Reactivation 워크플로우 트리거에 `Tag Added: 이번주-수신완료` 또는 `Tag Contains: 이번주-수신완료 AND Last Email Opened > 60 days` 조건을 추가[^9]. 중복 방지 3조건(수신거부·전환·지난 7일) 함께 심기. 6. 테스트 리드로 1회 체인 확인 — 가상 테스트 리드 1명을 워크플로우 A에 강제 진입시켜 (Contacts 탭 → 개별 리드 → Workflows → Add to Workflow) 태그가 붙는지 확인. 태그가 붙은 직후 Reactivation 트리거가 감지되는지 Workflow History(실행 이력) 화면에서 확인합니다.

팀 단위 · 6단계 조감

  • 01 0️⃣1️⃣ 팀 산출물 열기 + 지목 →
    0단계: 파이프라인 문서·워크플로우 목록·클라이언트 카드 열기. 1단계: 연결 지점 3개 빨간펜 표시. 순서가 중요합니다.
  • 02 2️⃣3️⃣ 태그 표준 + 액션 추가 →
    2단계: 태그 이름 3개 합의. 3단계: 18강 예약 워크플로우 말미에 Add Contact Tag 액션 추가 후 Draft 저장.
  • 03 4️⃣5️⃣ 트리거 갱신 + 체인 확인 →
    4단계: 20강 Reactivation 워크플로우 트리거에 Tag Contains 조건 추가 + 중복 방지 3조건. 5단계: 테스트 리드 1명으로 체인 1회 확인.

설정 시점에 · 이미 심기

  • 01 🚫 ① unsubscribed · 수신거부 →
    모든 워크플로우 트리거에 `Tag NOT contains "unsubscribed"`. 수신거부자에게는 어떤 경우에도 메시지가 가지 않는 1차 벽.
  • 02 ✅ ② converted · 전환 완료 →
    `Tag NOT contains "converted"`. 이미 전환된 리드에게 재활성화·DM 캠페인이 돌아가지 않도록 2차 벽.
  • 03 📅 ③ 지난 7일 · 수신 이력 →
    `Tag NOT contains "이번주-수신완료-지난7일"` + Time-based Tag Removal(시간 기반 태그 제거 · 7일 뒤 자동 제거) 액션 조합.
CONCEPT · 02
📝 강사 노트
💬 그대로 읽으세요
1. 병합 아키텍처 요약 1장 — 23강 파이프라인 위에 24강에서 추가한 태그 연결선 3개를 빨간 화살표로 그린 이미지 또는 요약 텍스트. 팀 드라이브·노션에 저장. 2. GHL 워크플로우 스크린샷 2장 — (a) 태그 추가 액션이 들어간 워크플로우 A Draft 화면 (b) 태그 기반 트리거가 들어간 워크플로우 B Draft 화면.

슬랙 `#수업실습`에 `[24강][A/B/C/D] 병합아키텍처·연결태그3종` 형식으로 업로드해 주세요. 실습 중에도 진행 화면을 자유롭게 공유해 주시면 서로 참고가 됩니다.

💬 그대로 읽으세요
1. 강사가 전면 화면에서 워크플로우 A 말미에 Add Contact Tag 액션 추가 → 워크플로우 B 트리거에 Tag Contains 조건 추가까지 한 사이클을 시연합니다. 2. 시연 후 팀별로 자기 23강 파이프라인에서 같은 작업을 2~3개 연결에 반복합니다. 3. 테스트 리드 1명으로 체인 1회 작동 확인. 4. 산출물 2점 슬랙 `#수업실습` 업로드. 5. 막히는 팀은 `#강사-QnA` 채널 또는 손 들기로 강사·운영진 호출.

태그 3종 표준 합의 → Add Contact Tag 액션 삽입 → Tag Contains 트리거 조건 갱신이 오늘의 병합 사이클입니다.
