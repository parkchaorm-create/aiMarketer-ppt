Purpose 한 문장 · 문서 네 블록

  • 01 🎯 Purpose · 템플릿 한 줄 →
    "[대상]에게 [관계 목표]를 [단계 흐름]으로 움직인다." A 푸르트 예: "3~7세 자녀 엄마에게 정기 구독 고객이 되도록 DM→상담→결제→수령→재구매."
  • 02 📐 문서 블록 1·2 · 목적·조감도 →
    블록 1은 Purpose 문장. 블록 2는 3레이어 조감도 — 태그 체계·단계 이름·Workflow 요지를 한 줄씩.
  • 03 📎 문서 블록 3·4 · 출처·이월 →
    블록 3은 재료 출처(ghl-deploy·브랜드 매뉴얼·클라이언트 카드). 블록 4는 24강 입력 + 25강 검증 포인트.
  • 04 🖼️ 도형 · 선택 · 글만도 충분 →
    입력·처리·출력 세 네모 + 화살표만 있어도 충분. 노션·미리캔버스·구글 슬라이드·손그림 사진 모두 가능합니다.

표 세 개 · 24강 입력 자료

  • 01 🗺️ Pipeline 1개 · 이름·단계 5 →
    이름({팀명}_신규문의_파이프라인)과 단계 3~5개(신규문의·상담·결제·납품·재구매)를 확정합니다.
  • 02 🏷️ Tag 2~3개 · 추가·제거 조건 →
    신규문의(랜딩 폼 제출 시 추가·상담 완료 시 제거)·vip·재활성화대상 중 2~3개. 각 태그 조건은 한 줄씩 명시.
  • 03 ⚡ Workflow 1개 · 트리거→액션 →
    트리거(랜딩 폼 제출) · 액션 1(태그 추가) · 대기(24h) · 액션 2(SMS) · 종료 조건(상담 예약 완료) 순으로 표 채움.
  • 04 ⏰ 시간 부족 시 · Pipeline·Tag 우선 →
    시간이 부족하면 Workflow 명세는 24강으로 이월. Pipeline + Tag 두 노드를 먼저 확정하세요.
