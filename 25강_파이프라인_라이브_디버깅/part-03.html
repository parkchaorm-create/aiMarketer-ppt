  • 01 🔁 1단계 · 재현 · 같은 조건 →
    같은 조건으로 한 번 더 실행. Claude Code는 `/resume`(슬래시 리줌 · 과거 세션 재개)으로 직전 세션 복원 · GHL은 Test Workflow 재실행.
  • 02 🔍 2단계 · 로그 · 실패 노드 →
    GHL은 Automation → Executions(실행 내역) 창에서 실패 노드 확인. Claude Code는 `--debug`(상세 로그 모드) 플래그로 API·도구·타이밍 상세 로그.
  • 03 💡 3단계 · 가설 · 플랜 모드 →
    Plan Mode 진입(`/plan` 또는 Shift+Tab 두 번) → Claude가 수정 계획만 읽기 전용으로 제시. "원인 후보 1·2·3" 한 문장씩 적기.
  • 04 🔧 4단계 · 수정 · 한 가지만 →
    가장 검증 쉬운 가설부터 한 번에 한 가지만 고치고 1단계 재현으로 돌아갑니다. 한 번에 두 가지 고치면 원인을 영원히 놓칩니다.

같은 조건으로 한 번 더 실행해 봅니다. Claude Code에서는 `/resume`(슬래시 리줌 · 과거 세션을 불러와 이어서 작업하는 슬래시 명령)으로 직전 테스트 세션을 그대로 복원할 수 있어요[^T1-3]. GHL은 `Test Workflow` 버튼으로 같은 컨택트·같은 워크플로우 재실행.

GHL에서는 워크플로우 실행 내역(좌측 메뉴 Automation → Executions)에서 실패한 노드가 표시됩니다. Claude Code는 `--debug`(디버그 · API 호출·도구 실행·타이밍을 상세 로그로 출력하는 커맨드라인 옵션) 플래그로 실행 시 상세 로그 획득[^T1-4][^T2-1].

이 단계가 핵심입니다. 곧바로 수정하지 않고 "이것 때문일 것 같다" 를 한 문장으로 적습니다. Claude Code의 플랜 모드(Plan Mode · 읽기 전용 분석 모드) 가 여기서 강력합니다. `/plan` 또는 Shift+Tab을 두 번 눌러 진입하면, 파일과 파이프라인을 읽기만 하고 수정은 안 하는 모드로 바뀝니다[^T1-5][^T2-2]. Claude가 "어디를 고칠지 계획만" 제시합니다.

가설 중 가장 검증이 쉬운 것부터 한 번에 한 가지만 고치고, 1단계 재현을 반복합니다. 한 번에 두 가지를 고치면 무엇이 진짜 원인이었는지 놓칩니다.
