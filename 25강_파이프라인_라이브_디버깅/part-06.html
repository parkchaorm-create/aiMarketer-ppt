남의 실패가 · 가장 좋은 · 교재일까요?

  • 01 🎓 대표 사례 · 교재가 됩니다 →
    에이전시 실무 교육에서 남의 실패 사례가 가장 좋은 교재입니다. 대표 팀이 된 실패 = 창피함이 아니라 공로.
  • 02 🗣️ 강사 · 소리 내어 · 4단계 →
    강사가 "지금 1단계 재현", "지금 2단계 로그"를 명시하며 시연합니다. 수강생은 단계 라벨을 따라 메모 한 줄씩.
  • 03 🔁 내 것에 · 한 줄 · 적용 →
    마지막 한 줄은 "내 파이프라인에 적용할 구체 액션". 노드명·조건명을 구체적으로 적으셔야 내 것이 됩니다.

03 / 07

관전 흐름 · 메모 5줄

  • 01 🎯 Step 1 · 대표 팀 · 재현 →
    강사가 재현 가능한 실패 하나를 지목 → 대형 화면에 공유 → Test Workflow 재실행 "지금 1단계입니다". 메모 1행.
  • 02 🔍 Step 2 · 로그 · Executions →
    GHL Executions 창 실패 노드 확인 + Claude Code `--debug` 로그 병행. "지금 2단계" 멘션 후 메모 2행.
  • 03 💡 Step 3 · 가설 · Plan Mode →
    Claude Code에서 `/plan` 또는 Shift+Tab 두 번 → 수정 계획 읽기 전용 제시. "지금 3단계" 후 메모 3행.
  • 04 🔧 Step 4~6 · 수정·재실행·질문 →
    일반 모드 복귀 → 가설 1개만 수정 → 1단계 재현. 결과 확인 후 수강생 질문 1~2개. 메모 4행 + 적용 액션 1행 = 총 5줄.

04 / 07

수정 1건 · 누적 10줄

  • 01 💾 백업 먼저 · Duplicate →
    GHL 워크플로우 우상단 `···` → `Duplicate`(복제) → 이름 `{기존이름}_backup_25강`. 백업 없이 수정 금지.
  • 02 🔬 Plan Mode → 한 곳만 →
    Claude Code `/plan` 진입 → "이 가설 검증하려면 어디 어떻게 바꿔야?" 질문 → 일반 모드 복귀 → 워크플로우에서 한 곳만 수정.
  • 03 📊 Before/After · 1줄 표 →
    케이스 번호·증상·수정한 곳(노드명)·가설→결과. 미해결도 기록 OK. 이 표가 `ghl-deploy.md` 규칙 갱신의 재료.
  • 04 📚 Phase 4 · 누적 자산 10줄 →
    16~25강 산출물 · 저장 위치 · 책임자 10줄. 팀 인수인계 문서이자 Phase 6 포트폴리오 사이트 실적 섹션 재료.
