💬 그대로 읽으세요
파이널 3부작은 한 번에 끝나는 수업이 아닙니다. 세 회차가 한 장으로 묶여 있어요.

오늘이 첫 편입니다. 껍데기만 만드는 날이에요. 홈 화면 대표 이미지가 완벽하지 않아도, 서비스 페이지 문구가 다듬어지지 않아도 "일단 URL이 라이브 상태" 가 합격선입니다. 35강에 콘텐츠를 채우고, 36강에서 리허설할 시간이 있습니다.

오늘 만든 URL은 36강 피칭 데이에서 실제 클라이언트에게 보내질 그 URL입니다. 37강 최종 피칭 데이의 발표 자료 마지막 페이지에도 이 URL이 들어갑니다. 오늘의 한 줄이 3주 뒤 피칭 무대의 한 줄이 되는 거예요.

  • 01 🏗️ 34강 오늘 · 껍데기 만들기 →

  • 03 🎤 36강 · 집들이 예행연습 →
    피칭 리허설 + 파이프라인 최종 점검. 오늘 URL이 36강 피칭에서 · 37강 최종 발표에서 그대로 쓰입니다.

📝 강사 노트
💬 그대로 읽으세요
오늘 수업은 새로 만드는 수업이 아닙니다. 이미 쌓아 둔 재료를 한 장의 웹사이트에 모으는 수업이에요. 각 팀이 아래 3가지를 손에 쥐고 있는지 먼저 확인합니다.

웹사이트 한 장을 팀 3~4인이 동시에 만들려면 역할 분담이 필수예요. 파이널 3부작 내내 유지할 3역할을 지금 정합니다. RACI(알·에이·시·아이 · Responsible·Accountable·Consulted·Informed · 역할 분담의 국제 표준 표기)를 간소화한 3역할 구조입니다.

Designer · Integrator · Content

  • 01 🎨 Designer · R · Stitch 시안 →
    R(Responsible · 실행 책임) — Stitch에서 4~5페이지 시안 생성 · 텍스트·컬러 수정. 쓰는 도구는 Stitch 단 하나.
  • 02 ⚙️ Integrator · A · Claude Code →
    A(Accountable · 승인 책임) — Claude Code에서 HTML 수정 · GHL 임베드 삽입 · Git 커밋·푸시 · Web Deploy(웹 배포).
  • 03 📦 Content · C · 재료 공급 →
    C(Consulted · 자문 책임) — 4·9·32강 재료 공급 · 최종 검수. 구글 드라이브와 팀 공유 문서가 주 활동 무대입니다.
  • 04 👥 3·4인 팀 · 배분 원칙 →
    3인이면 1인 1역할 · 4인이면 Designer 2인으로 분할. I(Informed · 공유 대상)는 팀 전원 · 슬랙 #수업실습으로 진행 공유.
