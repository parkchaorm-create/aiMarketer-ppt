💬 그대로 읽으세요
실습 B의 목표는 단 하나입니다 — 라이브 URL을 만드는 것. 실습 A에서 모은 HTML export 파일 · GHL 임베드 코드 · GitHub 저장소를 Claude Code가 하나로 연결합니다. 이 파트의 합격선은 팀 슬랙에 `https://{계정명}.github.io/portfolio/` 형식의 URL이 올라가는 것입니다.

1. 프로젝트 폴더로 이동: `cd ~/portfolio-{팀이름}/` 2. Stitch export HTML 복사: export 폴더 내용을 프로젝트 폴더에 배치 · `index.html` 기준 3. Claude Code 실행: `claude` 명령으로 터미널 진입 4. 플랜 모드 진입: `/plan` 입력 · Claude에게 "이 HTML을 어떻게 수정할지 계획만 세워 주세요" 요청 5. 편집 모드 복귀 · 한국어 교체: 영어 placeholder 텍스트를 팀 한국어 콘텐츠로 교체 6. GHL 임베드 삽입: Contact 페이지 또는 홈 하단에 Content가 준비한 iframe 코드 삽입 7. Git 초기화·커밋: `git init` · `git add .` · `git commit -m "feat(34강): {팀명} 포트폴리오 초기 배포"` 8. GitHub 원격 연결·푸시: `git remote add origin {저장소 URL}` · `git push -u origin main` 9. GitHub Pages 활성화: 저장소 Settings > Pages > Branch `main` 선택 > Save 10. 라이브 URL 확인: 잠시 대기 후 `https://{계정명}.github.io/portfolio/` 접속 (최초 배포는 짧은 지연이 정상)

Claude Code의 플랜 모드(`/plan` · 읽기 전용 분석 모드)가 오늘 가장 큰 안전장치입니다. 이유는 이래요.

Stitch export HTML은 생소한 구조일 수 있습니다. Tailwind 유틸리티 클래스가 수백 개 붙어 있고, 이미지 경로가 플레이스홀더인 경우도 많아요. 곧바로 편집 모드로 들어가 "이거 고쳐 줘, 저거 고쳐 줘"를 반복하면 의도치 않은 곳이 함께 바뀝니다. 이때 플랜 모드로 먼저 "어느 파일의 몇 번째 줄을, 왜, 무엇으로 바꿀지" 계획만 받아 보는 것이 안전합니다[^C1-1][^C1-2].
