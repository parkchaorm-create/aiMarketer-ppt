05 / 07

왜 Firebase Studio 아닌 · GitHub Pages

  • 01 ♻️ 계정 재사용 · 9·22강에서 이미 →
    GitHub은 9·22강에서 이미 가입·사용 경험이 있습니다. 새 계정 만들 시간이 수업 안에서 녹습니다.
  • 02 💸 무료 · 퍼블릭 100GB 대역폭 →
    GitHub Pages 퍼블릭 저장소는 월 100GB까지 무료. Firebase Studio는 일부 기능에 결제 카드 등록이 필요할 수 있음.
  • 03 🤝 Claude Code 적합성 · Git 기반 →
    Claude Code가 가장 잘 다루는 배포 경로가 Git push 기반. 오늘 흐름과 자연스럽게 맞물립니다.
  • 04 📎 Firebase · 36강 이후 심화 탐색 →
    Firebase Studio 경로도 공식 지원. 36강 리허설 이후 심화 탐색 재료로 더 적합합니다.

심화 궁금증에 답하는 박스 하나. Stitch는 공식적으로 Firebase Studio(파이어베이스 스튜디오 · Google의 클라우드 개발 환경)로도 바로 export 가능합니다[^S2-2]. 그런데 오늘 수업은 GitHub Pages를 표준 경로로 고정했습니다. 이유는 세 가지입니다.

1. 계정 재사용: GitHub은 9·22강에서 이미 가입·사용 경험이 있습니다. 신규 계정 생성 시간이 수업 안에 녹습니다. 2. 비용: GitHub Pages 퍼블릭 저장소는 월 100GB 대역폭까지 무료[^G2-1]. Firebase Studio는 일부 기능에 결제 카드 등록이 필요할 수 있습니다. 3. Claude Code 적합성: Claude Code가 가장 잘 다루는 배포 경로가 Git push 기반입니다.

3도구 연결 한 문장: Stitch 시안 → Claude Code 수정·배포 → GHL 임베드 삽입.
