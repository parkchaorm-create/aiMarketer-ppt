💬 그대로 읽으세요
실습 A에서는 각 팀의 Designer가 선택한 가상 클라이언트를 기준으로 Stitch에서 4~5페이지 시안을 생성합니다. 이 시안의 HTML export 파일이 실습 B의 입력이 됩니다. 동시에 Integrator는 Claude Code 환경을 점검하고, Content는 GHL 임베드 코드를 준비합니다. 세 역할이 병렬로 움직이는 구조입니다.

1. Stitch 접속: <https://stitch.withgoogle.com/> → 구글 계정 로그인 2. 프롬프트 설계: 4강 팀 브랜드 매뉴얼의 로고·컬러·폰트·슬로건을 한 단락으로 정리 3. Vibe Design 요청: "포트폴리오 웹사이트 4페이지를 만들어 주세요 — Home · Works · Services · Contact" 형태로 입력 4. 시안 검토·수정: 캔버스에서 텍스트·컬러 미세 조정 (보이스 명령 또는 텍스트) 5. HTML export: 우상단 export 버튼 → HTML/CSS (Tailwind) 선택 → 다운로드 6. 팀 공유 폴더에 저장: 구글 드라이브 또는 슬랙 `#수업실습` 채널에 업로드
