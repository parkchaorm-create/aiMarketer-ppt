📝 강사 노트
💬 그대로 읽으세요
Claude Code에는 이걸 구조적으로 지원하는 기능이 있어요. 서브에이전트(Subagent · 별도 컨텍스트로 작업을 위임받는 AI 분신) 입니다[^T1-3]. 오늘은 개념만 소개하고, 실습은 23강부터(팀 프로젝트 1/3) 본격 다룹니다.

분리된 책상에서 · 일하는 AI

  • 01 📁 .claude/agents/ · 파일로 정의 →
    .claude/agents/ 폴더에 .md 파일로 정의. 각 파일이 자기만의 시스템 프롬프트·허용 도구·모델을 가집니다.
  • 02 🪟 독립 컨텍스트 윈도 →
    메인 세션과 분리된 책상에서 실행. 도구 권한도 파일별로 제한 가능. 작업 후 요약만 메인에 복귀합니다.
  • 03 💬 호출 · "서브에이전트로 해 줘" →
    메인 세션에서 "뉴스레터 검수는 서브에이전트로 해 줘" 한 줄이면 위임 완료. 메인 컨텍스트는 깨끗하게 유지.
CONCEPT · 01
📝 강사 노트
💬 그대로 읽으세요
1. 조사(research)가 긴 작업: 여러 파일을 읽으며 결론만 필요한 경우 — 메인 컨텍스트를 오염시키지 않기 위해[^T1-2] 2. 도구 권한을 좁혀야 하는 작업: 예약 발행처럼 외부에 영향을 주는 작업은 서브에이전트에 제한된 권한만 부여 3. 관점이 달라야 하는 작업: 작성자와 검수자를 분리하면 메인이 자기 글에 편향되지 않음 (writer/reviewer 패턴)[^T1-2]

이번 실습부터 터미널 안에서 바로 쓸 수 있는 명령 두 개를 소개합니다. 외우지 않으셔도 되고, 필요할 때 꺼내 쓰시면 됩니다.

04 / 07

서브에이전트 · 적합한 세 장면

  • 01 🔍 조사 긴 작업 · 메인 오염 방지 →
    여러 파일을 읽으며 결론만 필요한 경우 — 메인 컨텍스트를 오염시키지 않기 위해 분신에게 맡깁니다.
  • 02 🔒 도구 권한 · 좁혀야 하는 작업 →
    예약 발행처럼 외부에 영향 주는 작업은 서브에이전트에 제한된 권한만 부여. 실수로 전체 계정 건드리는 일 방지.
  • 03 🎭 관점 분리 · 작성 vs 검수 →
    작성자와 검수자를 분리하면 메인이 자기 글에 편향되지 않음. 공식 문서의 writer/reviewer 패턴입니다.
CONCEPT · 02
📝 강사 노트
💬 그대로 읽으세요
현재 세션에 로드된 CLAUDE.md와 규칙 파일 목록을 보여주고, 클릭으로 해당 파일을 편집기에 열어 줍니다[^T1-1]. 오늘 실습에서 CLAUDE.md를 고치고 나서 "제대로 로드됐는지 확인" 하는 용도로 씁니다.

대화가 길어져 책상(컨텍스트 윈도)이 차오르기 시작하면, Claude Code가 스스로 지금까지의 대화를 요약해 공간을 비워 줍니다[^T1-1]. 중요한 점은 `/compact` 후에도 프로젝트 루트 CLAUDE.md는 자동으로 다시 읽힌다는 것[^T1-1]. 즉, 긴 실습 끝에 `/compact` 를 쳐도 오늘 작성한 본체 지시문은 살아남습니다.
