왜 고도화인가 · 카드 3장

  • 01 🖥️ 컨텍스트 윈도 · 책상 면적 →
    컨텍스트 윈도(Context Window · AI가 한 번에 기억하는 작업 책상 면적)의 한계를 비유로 이해합니다.
  • 02 💸 반복 입력 · 두 가지 비용 →
    같은 지시를 반복하면 사람 시간과 AI 토큰(AI가 읽는 글자 단위)이 이중으로 소모됩니다. 구체적 숫자로 보여드립니다.
  • 03 📦 22강 범위 · 3단 고도화 →
    본체 슬림화 + @imports + .claude/rules/. 세 단계로 쪼갠 구조의 전체 지도를 먼저 훑습니다.

02 / 07

CLAUDE.md가 500줄이 되면요?

  • 01 📏 Anthropic 공식 · 200줄 이하 →
    Anthropic 공식 문서: CLAUDE.md는 200줄 이하 목표. 길어질수록 컨텍스트를 더 소모하고 지시문 준수율이 떨어진다고 명시합니다.
  • 02 🎯 커뮤니티 · 500 토큰 이내 →
    커뮤니티 출처는 더 엄격하게 약 500 토큰(한글 300자 안팎) 이내를 권고합니다. 숫자는 달라도 방향은 같아요.
  • 03 📉 길수록 · 지시를 덜 따름 →
    10강 초안이 한 장이었다면 지금은 몇 배로 늘어났을 텐데요. 그대로 두면 AI가 지시문 절반을 잊기 시작합니다.

CONCEPT · 컨텍스트 윈도
03 / 07

책상 면적은 · 정해져 있습니다

  • 01 🖥️ 한 대화 · 한 책상 →
    Claude Code가 한 번의 대화에 쓸 수 있는 작업 책상 면적이 정해져 있습니다. 이걸 컨텍스트 윈도라고 부릅니다.
  • 02 📚 책상 위 · 여섯 가지 동시 →
    CLAUDE.md 본체 + @imports 파일 + rules 규칙 + 방금 친 프롬프트 + AI가 읽은 파일 + 대화 기록이 한 책상 위에 동시에 놓입니다.
  • 03 🗺️ 대형 지도 · 항상 펴 둘 일 X →
    CLAUDE.md를 무작정 늘리는 건 책상 절반을 대형 지도로 덮는 것과 같습니다. 지도는 필요할 때만 펴는 게 맞습니다.

CONCEPT · 반복 비용
04 / 07

같은 말 반복 · 두 배로 샙니다

  • 01 👤 사람 비용 · 매번 10~30초 →
    "우리 톤은 담담하게…"를 매 세션 입력하는 시간. 한 달 단위로 보면 업무 시간의 무시 못할 조각이 됩니다.
  • 02 🤖 AI 비용 · 매 요청마다 소모 →
    같은 지시가 토큰(Token · AI가 텍스트를 쪼개 읽는 단위)으로 재전송되어 매 요청 컨텍스트가 같은 양만큼 사라집니다.
  • 03 🪁 한 번 쓰면 · 두 비용 동시 절감 →
    CLAUDE.md에 한 번 적어 두면 사람은 입력하지 않고, AI는 자동으로 읽고, 책상 공간은 필요한 만큼만 씁니다.

05 / 07

오늘 · 세 가지 기법만 얹습니다

  • 01 📝 본체 슬림화 · 30~60줄 →
    10강에서 만든 초안을 그대로 두고, 핵심 30~60줄만 본체에 남깁니다. 통째 재작성이 아닙니다.
  • 02 🔗 @imports · docs/ 분리 →
    길어지는 섹션은 docs/ 폴더의 별도 파일로 이동하고, 본체에 @docs/파일이름.md 한 줄만 남깁니다.
  • 03 📂 .claude/rules/ 신설 →
    GHL 배포 규칙을 .claude/rules/ghl-deploy.md 별도 파일로. 2026년 커뮤니티 합의 권고안입니다.
