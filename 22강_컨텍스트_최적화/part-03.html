쪽지 한 장 · 서재로 넓어져요?

  • 01 📜 10강 · 한 장짜리 쪽지 →
    10강의 CLAUDE.md는 한 장짜리 쪽지. 프로젝트 개요·클라이언트·폴더 정도로 충분히 담겼습니다.
  • 02 📚 22강 · 쪽지 + 책장 + 서랍 →
    22강은 그 쪽지 + 옆 책장 몇 칸(@imports) + 서랍 하나(.claude/rules/)로 된 작은 서재가 됩니다.
  • 03 🔍 검색 · 수정 단위가 작아짐 →
    파일 수가 늘어나는 건 복잡해지는 게 아닙니다. 옷장이 서랍 칸으로 나뉘는 것과 같은 정돈이에요.

한 파일 · vs · 네 파일

  • 01 📄 Before · 본체 150~400줄 →
    10~21강 누적으로 한 파일 본체가 길어졌습니다. 가독성 낮음 · AI 준수율 낮음 · 유지보수는 전체 훑기.
  • 02 📚 After · 4~6파일 · 본체 30~60줄 →
    After에는 본체 슬림 + docs 2개 + rules 1~3개. 목차처럼 한눈에 파악되고 핵심이 상단에 집중됩니다.
  • 03 🔧 수정 · 해당 파일만 열면 됨 →
    뉴스레터 톤을 고치고 싶다면 docs/newsletter-voice.md 한 장만 열면 됩니다. 본체는 건드리지 않아도 돼요.

04 / 07

본체 · 여섯 덩어리만 남기기

  • 01 🎯 30초 안에 · 모양 파악 →
    본체에는 프로젝트를 처음 만난 사람이 30초 안에 이해해야 할 것만 남깁니다. 공식 문서는 "프로젝트의 모양만 담고 상세는 연결"로 요약합니다.
  • 02 📋 6덩어리 · 한 줄씩 →
    개요·선택한 클라이언트·주요 작업·폴더 구조·임포트 라인·민감 정보 금지 경고 — 각 한 줄~세 줄로 압축.
  • 03 📐 총량 30~60줄 기준 →
    30줄 미만은 정보 부족, 60줄 초과면 다음 섹션도 docs/로 분리. 이 범위가 오늘 실습 A의 합격선입니다.

05 / 07

@imports · vs · .claude/rules/

  • 01 🔗 @imports · 세션 시작 자동 →
    @docs/파일이름.md 한 줄이면 세션 시작 시 자동으로 끌어와 본체와 한 맥락. 최대 5단계 재귀 임포트 가능.
  • 02 📂 rules/ · 경로 한정 로드 →
    .claude/rules/ 폴더의 .md 파일은 YAML(야믈 · 설정 표기법) 프론트매터의 paths에 일치하는 파일을 만질 때만 활성화됩니다.
  • 03 ⚠️ 본체+임포트 합계 300줄 한도 →
    @imports로 끌어오는 파일도 결국 책상 위에 올라갑니다. 총량 300줄을 넘으면 효과 사라짐. rules/ 를 적극 활용하세요.

Part 03 · 세 줄 정리

  • 01 📚 목표 구조 · 본체 + @ + rules →
