실습 B · 한 장짜리 규칙 파일

  • 01 📂 .claude/rules/ · 폴더 신설 →
    터미널에서 mkdir -p .claude/rules. 기존 파일 덮어쓰지 않는 새 폴더라 안전합니다.
  • 02 📝 ghl-deploy.md · 5~7줄 본문 →
    17~20강 리드 태깅·예약 발행·재활성화 규칙을 선택한 클라이언트 기준으로 한국어 5~7줄 압축.
  • 03 🎯 YAML · 경로 한정 로드 →
    상단 YAML 프론트매터의 paths에 content/** · newsletters/** 글로브(glob · 경로 패턴) 지정. 해당 폴더 편집 시만 활성화.

규칙 · 항상 켜 두어야 할까요?

  • 01 💡 경로 한정 · 필요할 때만 →
    이 파일은 content/ · newsletters/ 폴더의 파일을 편집할 때만 자동 로드됩니다. 평소에는 책상 위에 없어요.
  • 02 🛟 새 파일 · 실패 안전망 →
    기존 CLAUDE.md와 다르게 새로 만드는 파일이라 기존 내용을 지우지 않습니다. 부담 없이 작성하셔도 됩니다.
  • 03 🔮 23강 재료 · 서브에이전트에 위임 →
    오늘 만든 한 장이 23강 서브에이전트 .claude/agents/ghl-operator.md 의 규칙 입력으로 사용됩니다. 오늘의 한 장이 내일의 재료.

6단계 · 폴더부터 로드 확인까지

  • 01 📁 ①~② 폴더 · 새 파일 생성 →
    터미널에 mkdir -p .claude/rules → 좌측 파일 탐색기에서 해당 폴더 우클릭 → New File → 이름 ghl-deploy.md.
  • 02 📋 ③~④ YAML · 본문 5~7줄 →
    상단 YAML 프론트매터 3줄(paths 글로브 3개) 붙여넣기 → 본문에 선택한 클라이언트 기준 규칙 5~7줄.
  • 03 💾 ⑤~⑥ 저장 · /memory 캡처 →
    Ctrl+S(Mac은 Cmd+S) 저장 → content/ 폴더 아무 파일 열고 다듬어 달라 요청 → /memory 실행 → ghl-deploy.md 목록 확인 캡처.

04 / 07

뼈대 · 네 섹션 · 복사해 쓰기

  • 01 📌 네 섹션 · 고정 순서 →
    리드 태깅 → 예약 발행 원칙 → 재활성화 → 뉴스레터 발송. 섹션 제목은 고정이고 내용만 클라이언트 기준으로 채움.
  • 02 🎨 톤 · 규칙 한 줄에 녹이기 →
    A 푸르트 = 담담·계절감, B 슬로우템포·C 손결·D 링크드마인드는 각자 톤 앵커. 한 줄 규칙에 정체성이 녹아야 재현성 확보.
  • 03 📝 예시 · A 푸르트 기준 →
    VIP = 주문 3회+·누적 15만원+ / 발행 = 화목 10시·일 18시 / 재활성화 = 90일 무반응 + "계절 바뀌었어요" 톤.

05 / 07

막히는 세 지점 · 먼저 짚기

  • 01 ⚠️ YAML · 파일 1줄째에 시작 →
    YAML 프론트매터의 --- 앞에 빈 줄이 있으면 인식 안 됨. 반드시 파일 맨 첫 줄에 시작해야 합니다.
  • 02 🔣 글로브 · 따옴표로 감싸기 →
    paths 아래 "content/**/*.md" 처럼 따옴표 권고. 공백 포함 경로에서도 안정성이 높아집니다.
  • 03 ⚙️ 점 누락 · rules/ X · .claude/rules/ →
    .claude/rules/ 의 점(.)을 빠뜨리면 일반 폴더가 되어 규칙으로 인식되지 않습니다. 점 잊지 마세요.
