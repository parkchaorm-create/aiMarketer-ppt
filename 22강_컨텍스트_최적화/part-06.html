실습 B에서는 `.claude/rules/` 폴더에 `ghl-deploy.md` 파일을 한 장 만듭니다. 17~20강에서 설정한 리드 태깅·예약 발행·재활성화 규칙을 선택한 클라이언트 기준으로 5~7줄의 한국어 규칙으로 요약합니다. 이 파일은 `content/` 또는 `newsletters/` 폴더의 파일을 편집할 때만 자동 로드됩니다[^T1-1].

1. `.claude/rules/` 폴더 만들기: 터미널에서 `mkdir -p .claude/rules` (없으면 생성, 있으면 그대로 통과). 2. 새 파일 만들기: 좌측 파일 탐색기에서 `.claude/rules` 폴더 우클릭 → `New File` → 이름 `ghl-deploy.md`. 3. 상단 YAML 프론트매터 붙여넣기:

--- paths:

"content/**/*.md"

"newsletters/**/*.md"

실습 B · 한 장짜리 규칙 파일

  • 01 📂 .claude/rules/ · 폴더 신설 →

  • 02 📝 ghl-deploy.md · 5~7줄 본문 →
    17~20강 리드 태깅·예약 발행·재활성화 규칙을 선택한 클라이언트 기준으로 한국어 5~7줄 압축.
  • 03 🎯 YAML · 경로 한정 로드 →
    상단 YAML 프론트매터의 paths에 content/** · newsletters/** 글로브(glob · 경로 패턴) 지정. 해당 폴더 편집 시만 활성화.

막히는 세 지점 · 먼저 짚기

  • 01 ⚠️ YAML · 파일 1줄째에 시작 →
    YAML 프론트매터의 --- 앞에 빈 줄이 있으면 인식 안 됨. 반드시 파일 맨 첫 줄에 시작해야 합니다.
  • 02 🔣 글로브 · 따옴표로 감싸기 →
    paths 아래 "content/**/*.md" 처럼 따옴표 권고. 공백 포함 경로에서도 안정성이 높아집니다.
  • 03 ⚙️ 점 누락 · rules/ X · .claude/rules/ →
    .claude/rules/ 의 점(.)을 빠뜨리면 일반 폴더가 되어 규칙으로 인식되지 않습니다. 점 잊지 마세요.
