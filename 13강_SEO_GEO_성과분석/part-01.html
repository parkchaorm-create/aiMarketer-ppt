반갑습니다. 13강 시작하겠습니다. 오늘의 주제는 SEO/GEO 성과 자동 분석이에요. 풀어서 말하면 "내가 쓴 글·운영 중인 페이지가 실제로 검색에서 보이고 있는지를 데이터로 확인하고, 그 결과를 리포트 한 장으로 자동 정리하는 절차"라고 이해해 주시면 됩니다. 6강에서 "검색에 잡히는 글의 원리"를 배우셨다면, 오늘은 그 글이 정말로 잡히고 있는지 를 확인하는 수업입니다.

오늘 수업 안에서 두 가지를 직접 만들어 봅니다. 첫째, URL 1개의 진단 리포트 1장 (실습 A). 둘째, URL 3개를 합친 합산 리포트 + 3개월 액션 플랜 (실습 B). 둘 다 Claude Code와 더미 데이터를 조합해서 만드실 거예요.

오늘 한 번 · 끝까지 · 돌려 보기

  • 01 📊 SEO·GEO · 한 문장 정리 →
    SEO(Search Engine Optimization · 검색 엔진 최적화)와 GEO(Generative Engine Optimization · 생성형 엔진 최적화)의 관계를 한 문장으로 설명합니다.
  • 02 📈 지표 5개 · 정의 확인 →
    GSC(Google Search Console · 구글 서치 콘솔)·GA4(Google Analytics 4)의 5대 지표(노출·클릭·CTR·평균 순위·참여율) 정의를 짚어 봅니다.
  • 03 📄 실습 A · URL 1개 리포트 →
    Claude Code의 WebFetch(URL을 받아 페이지를 분석하는 도구)와 더미 CSV로 URL 1개의 진단 리포트 한 장을 자동 생성합니다.
  • 04 📑 실습 B · URL 3개 합산 →
    같은 클라이언트 URL 3개를 병렬 분석해 합산 리포트 + 3개월 액션 플랜 + GEO 친화도 5문항 체크까지 마칩니다.

💬 그대로 읽으세요
6강에서 SEO의 기초 원리를, 10~12강에서 Claude Code로 콘텐츠 생산 라인을 익히셨어요. 그런데 여기서 빠진 조각이 하나 있습니다. "만든 콘텐츠가 정말로 성과를 내고 있는가?" 를 확인하는 단계가 없으면, 콘텐츠는 쌓이지만 개선이 안 됩니다.

1인 대행사 현실에서는 이 단계가 더 중요해요. 클라이언트에게 매달 "이번 달 어떤 성과가 있었습니까?"라는 질문을 반드시 받거든요. 그때 "느낌상 잘 되고 있어요"라고 답하면 재계약이 어렵습니다. 숫자로 답할 수 있어야 합니다.
