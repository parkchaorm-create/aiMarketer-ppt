💬 그대로 읽으세요
지금부터 선택한 클라이언트 기준 URL 1개의 진단 리포트 1장을 만드시게 됩니다. 손으로 정리하면 반나절 걸리는 작업이에요. 오늘은 Claude Code에 프롬프트 1개를 넣고 5~10분 안에 리포트 초안이 생성되는 것을 확인해 보겠습니다. 강사는 진행 흐름만 짚어 드리고, 막히는 분은 손 들어 주시면 옆에서 도와드릴게요.

실제 GSC/GA4 계정 없이도 진행 가능하도록, 클라이언트별 샘플 URL을 다음과 같이 가정하고 진행합니다. 더미 CSV 데이터와 이 URL의 실제 페이지 구조를 함께 분석합니다.

A · B · C · D · 샘플 URL 지도

  • 01 🌸 A 푸르트 · 소상공인 간판 →
    네이버 플레이스 또는 인스타그램 공식 링크. 소상공인 온라인 간판 유형. 검색 의도는 주로 "지역명+업종".
  • 02 🍽️ B 슬로우템포 · F&B 상세 →
    네이버 플레이스 · 블로그 개업 소개글. 식음료(F&B) 상세 페이지 유형. 체류 품질이 중요합니다.
  • 03 🏺 C 손결 · 공방 블로그 →
    블로그 대표 시리즈 1편(예: "흙 고르는 법"). 공방 블로그 콘텐츠 유형. 롱테일 키워드 유리합니다.
  • 04 🧠 D 링크드마인드 · 지식 서비스 →
    링크드인 장문 글 1편 · 뉴스레터 백호. 지식 서비스 랜딩 유형. E-E-A-T 점수 특히 중요합니다.

검토 · 5분 · 세 가지만

  • 01 🔢 포인트 1 · 숫자 = CSV 실제값? →
    리포트의 "핵심 지표" 표 수치가 CSV 파일의 실제 값과 같은지 대조. 다르면 그 칸만 직접 수정이 가장 빠릅니다.
  • 02 🎯 포인트 2 · 원인 가설 구체성 →
    "글이 짧음" 같은 막연한 표현이 아니라 "H2 소제목이 2개뿐" 같이 페이지 구조와 연결된 구체 가설인지 확인합니다.
  • 03 ⚠️ 포인트 3 · 할루시네이션 경고 →
    마지막 줄에 "AI 할루시네이션 경고: 숫자와 문장은 발송 전 사람이 확인 필요" 한 줄이 들어갔는지 확인합니다.
