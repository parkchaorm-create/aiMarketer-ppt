세 한계 · 세 전략

  • 01 🧠 한계 · AI는 얼굴을 기억 안 함 →
    Character Consistency(캐릭터 컨시스턴시 · 여러 컷에 같은 인물을 유지)가 어려운 근본 이유 세 가지를 먼저 짚습니다.
  • 02 🗝️ 전략 · Soul ID/참조/반복 →
    힉스필드가 제공하는 일관성 전략을 사용자 관점에서 Soul ID · 이미지 참조 · 프롬프트 반복 세 갈래로 단순화해 이해합니다.
  • 03 🎯 오늘 주경로 · 프롬프트 반복 →
    실습은 가장 쉬운 프롬프트 반복(Prompt Anchoring) 전략. 캐릭터 상세 묘사 블록 한 덩어리를 3컷에 그대로 복붙합니다.

왜 · 매번 얼굴이 바뀔까요?

  • 01 🎲 한 번 돌리면 · 다른 사람 →
    26강에서 같은 프롬프트를 두 번 돌려 보셨다면 이미 체감하셨을 겁니다. 비슷은 한데, 같은 사람은 아닌 얼굴이 나옵니다.
  • 02 ❓ 왜 다른가요 · 이유 3개 →
    이 질문에 답을 못 하면 27강 전체가 안 풀립니다. "왜"를 알면 "어떻게 붙잡는지"도 자연스럽게 이어집니다.
  • 03 🔗 이해 → 전략 · 연결고리 →
    세 한계 각각에 하나씩 대응 전략이 붙습니다. Part 02 이론 → Part 03 시연 → Part 05 실습 흐름의 지도입니다.

CONCEPT · 한계 3가지
03 / 07

확률적 결과물의 세 한계

  • 01 🧠 한계 1 · 얼굴을 기억 안 함 →
    AI는 특정 얼굴을 저장하지 않습니다. "30대 여성·단발·베이지 블라우스" 같은 추상적 특징만 매번 다시 조합합니다.
  • 02 🎲 한계 2 · 독립된 샘플링 →
    매 생성은 독립 샘플링입니다. 힉스필드 공식도 "같은 프롬프트·같은 모델도 결과가 완전히 같지 않다"고 명시합니다.
  • 03 📺 한계 3 · 방치하면 광고 불가 →
    3컷 릴스에 주인공이 서로 다른 얼굴이면, 고객은 "같은 브랜드 맞나?" 하고 혼란을 느낍니다. 해결해야 브랜드 영상이 됩니다.

CONCEPT · 일관성 3전략
04 / 07

세 전략 · 강·중·쉬움

  • 01 💪 Soul ID · 가장 강력 →
    Soul ID(소울 아이디 · 힉스필드의 캐릭터 학습 기능)는 인물 사진 20장+ 업로드하고 5분 학습하면 같은 얼굴을 고정합니다. 유료 성향.
  • 02 🖼️ 이미지 참조 · 중간 난이도 →
    Reference(레퍼런스 · 참조 이미지) 1장을 업로드해 "이 사람 기반"으로 만드는 방식. 포즈·표정이 결과에 크게 반영됩니다.
  • 03 🔁 프롬프트 반복 · 오늘 주경로 →
    Prompt Anchoring(프롬프트 앵커링 · 같은 묘사 블록 반복 삽입). 캐릭터 상세 묘사 한 덩어리를 3컷에 똑같이 복붙합니다.
  • 04 🎯 오늘은 · 3번 전략만 →
    Soul ID는 강사 시연만, 오늘 수강생 실습은 프롬프트 반복 하나로 통일합니다. 20장 사진 준비 부담과 크레딧 소모 때문입니다.

RECAP · 복습
05 / 07

Part 02 · 세 줄 정리

  • 01 🧠 얼굴 기억 안 함 · 매번 조합 →
    AI는 얼굴을 저장하지 않고, 추상 특징만 매번 확률적으로 다시 조합합니다. 그래서 같은 프롬프트에도 얼굴이 흔들립니다.
  • 02 🗝️ 세 전략 · 강·중·쉬움 →
    Soul ID(강·유료 성향) · 이미지 참조(중) · 프롬프트 반복(쉬움). 난이도와 결과 안정성이 반비례 관계는 아닙니다.
  • 03 🔁 오늘 · 프롬프트 반복 주경로 →
    캐릭터 상세 묘사 블록을 3컷에 반복 삽입하는 방식. Soul ID는 Part 03에서 강사 시연만 보시면 됩니다.
