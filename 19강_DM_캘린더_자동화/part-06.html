지금까지 만든 두 조각을 한 줄로 잇는 작업입니다. "DM이 도착하면 → 첫 응대 메시지 + 버튼 → 버튼 클릭 시 예약 페이지 열림 → 예약 완료 시 리마인더 흐름 시작". 새로 만드는 것은 없습니다. 두 워크플로우를 버튼 한 개로 잇는 것이 오늘 남은 과제예요.

  • 01 📅 Simple Calendar · 5단계 →
    캘린더 생성 → 기본 설정 → 예약 후 자동화 연결 → 리마인더 2개 → 공개 URL 발급. 다섯 단계로 예약 페이지가 완성됩니다.
  • 02 ⏰ 24h SMS · 1h Email →
    Wait(대기) 액션 + 메시지 액션 조합으로 24시간 전 SMS · 1시간 전 Email 두 번의 리마인더를 배치합니다.
  • 03 🔗 연결 · 실습 C · 버튼 한 개 →
    실습 A의 예약하기 버튼에 실습 B의 공개 URL을 연결. 두 흐름이 버튼 한 개로 이어지는 마지막 조각입니다.

📝 강사 노트
💬 그대로 읽으세요
1. 실습 A 워크플로우로 되돌아가기 — Automation → Workflows → 실습 A에서 만든 DM 자동 회신 워크플로우 열기. 2. 버튼 후속 액션 지정 — `예약하기` 버튼의 "On Click" 후속 액션에, 실습 B에서 만든 예약 페이지 공개 URL을 메시지로 보내는 액션을 추가. 같은 순간 `high-intent` 태그를 추가하는 액션도 함께 배치. 3. 최종 테스트 — Draft 상태에서 가상 고객으로 전체 흐름 1회 통과. DM 테스트 → 버튼 클릭 → 예약 페이지 접속 → 가상 예약 → 리마인더 발송 확인. 확인되면 Publish.

본 블록은 희망자만 따라 하는 선택 심화입니다. 요건을 충족하는 수강생만 진행하세요.

예약 확정 → 두 번의 리마인더

  • 01 🎯 트리거 · Customer Booked Appt →
    Customer Booked Appointment 트리거는 예약 완료 시점에 자동 기동됩니다. 태그 추가·메시지·파이프라인 이동 등 후속 액션을 붙일 수 있어요.
  • 02 ⏳ Wait Until · 시점 기준 →
    Wait Until(예약 시간 − 24시간) · Wait Until(예약 시간 − 1시간) 두 번의 대기 액션으로 리마인더 시점이 잡힙니다.
  • 03 💌 문구 3원칙 · 짧게 담백 →
    예약 정보 명시 · 취소·변경 링크 한 줄 · 이모지 남용 금지. 세 원칙만 지키면 담백한 리마인더가 됩니다.

📝 강사 노트
💬 그대로 읽으세요
개인 Google 계정에 대한 OAuth 인가 가능

1. 브라우저에서 `claude.ai/settings/connectors` 페이지 열기. 2. "Google Calendar" 커넥터를 찾아 추가 → Google 계정 인가 → 권한 확인. 3. 클로드 코드(대화형 세션)를 실행 후 "오늘 내 Google 캘린더 일정 목록을 알려 주세요" 같은 자연어 질문으로 조회 테스트[^10]. 4. 이어서 "내일 오후 3시에 가상 '푸르트 샘플 체험' 30분 일정 만들어 주세요"라고 요청해 생성 여부 확인. 5. 실제 사용자 계정에 흔적이 남으므로, 체험 후에는 해당 일정 수동 삭제.

Google Calendar MCP는 개인 일정 관리 범용 도구입니다. 오늘 만든 가상 클라이언트 응대 플로우는 이미 GHL 내장 캘린더에서 완결되어 있어요.

버튼 한 개 + (선택) MCP 감 체험

  • 01 🔗 실습 C · 연결 3단계 →
    실습 A 열기 → 예약하기 버튼 On Click에 공개 URL + 태그 추가 → 전체 흐름 1회 테스트 후 Publish. 새로 만드는 건 없습니다.
  • 02 🏷️ 태그 자동 추가 · 17강 잇기 →
    버튼 클릭 시 high-intent 태그를 함께 추가하는 액션을 배치하면 17강 태그 체계가 자동으로 유지됩니다.
  • 03 🌱 선택 · Google Cal MCP 감 체험 →
    claude.ai/settings/connectors에서 Google Calendar 커넥터 추가 후 자연어로 일정 조회·생성을 대화형 세션에서 1회 체험합니다.
  • 04 ⚠️ 주의 · 대화형 세션만 →
    v2.1.79 이후 비대화형 모드(-p · --resume)에서 일부 커넥터가 비활성화되는 이슈가 보고됐어요. 오늘은 대화형 세션에서만 진행합니다.

이 Demo의 교육 목적은 "클로드 코드가 로컬 시스템뿐 아니라 내 개인 생산성 도구에도 연결될 수 있다" 는 감 체험입니다. 20~25강에서는 다시 GHL이 주 무대가 됩니다.
