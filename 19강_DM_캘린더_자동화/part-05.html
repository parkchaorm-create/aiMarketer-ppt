6단계 · 한 호흡으로

  • 01 🔧 17강 재활용 · 트리거·액션만 교체 →
    지난 17강 폼 워크플로우와 구조가 같아요. 트리거 유형과 액션 유형 두 지점만 새로 바꿉니다.
  • 02 🎯 트리거 · 메시지 수신 →
    Instagram Message Received 또는 Facebook Message Received. 연동 안 된 분은 Customer Replied(웹챗 위젯)로 대체합니다.
  • 03 ⚙️ 액션 · Interactive Messenger →
    버튼 · 퀵리플라이(Quick Reply · 미리 준비된 응답 옵션)가 포함된 구조화 메시지 액션으로 경로를 나눕니다.

HOOK · 첫 응대의 감각
02 / 07

"곧 답드릴게요" · 그 한 줄

  • 01 ⏱️ 첫 반응 · 60초 안 →
    고객이 DM을 보낸 뒤 60초 안에 무언가 돌아오는 경험이 가장 큰 신뢰 신호가 됩니다.
  • 02 🔘 버튼 2~3개 · 친절한 선택 →
    예약하기 · 자료 받기 · 상담원 연결 같은 버튼 2~3개로 고객이 스스로 다음 행동을 고르게 합니다.
  • 03 🎨 톤 · 브랜드 말투 유지 →
    응대 메시지는 선택한 가상 클라이언트의 브랜드 톤(담백·차분·느린 호흡 등)에 맞춰 한 줄씩 조정합니다.

CONCEPT · 분기 설계
03 / 07

태그 기준 · 두세 갈래

  • 01 🚀 high-intent · 예약 직진 →
    17강에서 붙인 high-intent(고관심) 태그 보유 고객은 예약 페이지 바로 안내하는 경로로 보냅니다.
  • 02 🌱 need-nurturing · 자료 먼저 →
    need-nurturing(양육 필요) 태그는 구매 전 정보가 더 필요한 리드. 예약보다 자료 전달 경로가 자연스러워요.
  • 03 🕊️ 그 외 · 기본 라우팅 →
    태그가 없거나 다른 태그만 있는 경우는 기본 안내(자기소개 + 선택지 3개)로 라우팅합니다.

CONCEPT · 클로드 코드 프롬프트
04 / 07

초안 · AI가 · 도와줍니다

  • 01 📋 복사용 프롬프트 · 대괄호만 교체 →
    실습 키트에 있는 프롬프트(Prompt · AI 지시문)를 그대로 복사하고 [대괄호] 부분만 본인 가상 클라이언트 정보로 교체하세요.
  • 02 🧠 클로드 코드 · 설계 초안 →
    분기별 응답 메시지 3~4줄 · 버튼 라벨 3개 · 후속 액션 한 줄 설명을 마크다운 형식으로 받아 볼 수 있어요.
  • 03 🖊️ GHL UI에 · 재료로 →
    받아 온 초안은 재료로만 사용합니다. 브랜드 톤과 맞지 않는 문장은 한두 줄 다듬어 GHL 메시지 입력 칸에 붙여 넣으세요.

RECAP · 자주 막히는 지점
05 / 07

막히면 · 먼저 · 여기 확인

  • 01 🕳️ 트리거 안 보임 · 연동 확인 →
    Instagram Message Received가 안 보이면 Settings → Integrations에서 IG/FB 연결 확인. 안 되면 Customer Replied로 우회.
  • 02 ⚫ 액션 회색 · 채널 활성화 →
    Interactive Messenger 액션이 회색이면 Location Settings → Messaging에서 해당 채널 활성화 여부 확인합니다.
  • 03 📣 Publish 누락 · 우측 상단 →
    테스트는 가는데 실제 DM이 안 오면 우측 상단 Save → Publish 순서로 전환되어 있는지 확인해 주세요.
  • 04 🔘 버튼 무반응 · On Click 지정 →
    버튼을 눌러도 아무 반응이 없으면 각 버튼 설정의 On Click 후속 액션(태그 추가·다음 단계) 지정 여부 확인합니다.
