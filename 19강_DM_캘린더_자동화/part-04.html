📝 강사 노트
💬 그대로 읽으세요
지난 17강의 폼 워크플로우 경험을 그대로 활용합니다. 다른 점은 트리거 유형과 액션 유형뿐이에요. 트리거는 `Instagram Message Received` 또는 `Facebook Message Received`, 액션은 `Instagram Interactive Messenger` 또는 `Facebook Interactive Messenger`. 공식 헬프에 따르면 이 액션은 버튼·퀵리플라이 같은 인터랙티브 요소를 포함한 메시지를 자동 발송합니다[^5].

실습 A는 6단계로 나뉩니다. 세부 단계는 노션 튜토리얼(`실습_튜토리얼/실습A_DM자동회신.md`)에 화면 캡처와 함께 정리되어 있어요.

01 / 07

두 길을 · 먼저 구분합니다

  • 01 🛣️ 주 경로 · GHL 내장 캘린더 →
    오늘 실습 A·B·C의 무대는 GHL 내장 Automation → Workflows + Calendars 화면입니다. 가상 클라이언트 응대 플로우의 주 경로입니다.
  • 02 🌱 선택 · Google Calendar MCP →
    개인 Google 캘린더에 일정을 생성·조회하는 클로드 코드 커넥터는 선택 심화 체험용입니다. 요건 충족 수강생만 진행해요.
  • 03 ✅ 사전 체크 · 4가지 →
    소셜 연동 · 샌드박스 번호 · 가상 클라이언트 선택 · 태그 5종 존재. 네 가지를 먼저 확인한 뒤 실습을 시작합니다.
OVERVIEW
📝 강사 노트
💬 그대로 읽으세요
1. 가상 클라이언트 재확인 — 1강에서 선택한 A/B/C/D 중 하나. 응대 말투와 영업 시간을 그 브랜드 톤에 맞춰 구성합니다. 2. Automation → Workflows → New Workflow (Create from Scratch) 3. Trigger 추가: `Instagram Message Received` 또는 `Facebook Message Received` 중 선택(소셜 계정 연동 상태에 따라). 연동이 안 된 분은 `Customer Replied`(웹챗 위젯 기반). 4. If/Else 조건 분기 추가: 17강에서 만든 태그 조건으로 2~3개 경로 구성 — 예: `high-intent` 보유 → 예약 안내 경로 / 그 외 → 자료 안내 경로. 5. Action 추가: 각 분기 끝에 `Instagram/Facebook Interactive Messenger` 액션 배치. 버튼 라벨 2~3개(예: `예약하기` · `자료 받기` · `상담원 연결`) 설정. 6. Save as Draft → 테스트 메시지 → Publish. `#수업실습` 채널에 `[19강][A/B/C/D] DM자동회신_링크` 형식으로 공유.

워크플로우 설계 초안을 얻고 싶을 때 아래 프롬프트를 클로드 코드 터미널에 그대로 붙여 넣으세요. [대괄호] 부분만 교체하시면 됩니다.

왜 · 두 무대를 · 구분할까요?

  • 01 🎭 무대 1 · 가상 클라이언트 응대 →
    고객 메시지 · 예약 확정 · 리마인더는 가상 클라이언트 브랜드 소유. GHL 안에서 완결되는 것이 자연스럽습니다.
  • 02 🙋 무대 2 · 내 개인 생산성 →
    Google Calendar MCP는 내 개인 일정 관리 범용 도구. 브랜드 운영이 아니라 내 생활에 쓰는 도구입니다.
  • 03 🧭 혼동 방지 · 역할 분리 →
    두 무대를 섞으면 개인 일정이 고객 응대에 묻거나 반대 상황이 생길 수 있어요. 세팅 단계에서 분리합니다.

실습 시작 전 · 네 가지 확인

  • 01 📱 소셜 연동 · IG · FB →
    Settings → Integrations에서 Instagram · Facebook 연결 상태 확인. 안 되어 있으면 웹챗 위젯 트리거로 우회합니다.
  • 02 📞 샌드박스 번호 · 강사 안내 →
    샌드박스(Sandbox · 격리된 테스트 환경) 전화번호는 강사가 채팅으로 공지합니다. 본인 실번호·타인 번호 입력 금지.
  • 03 👤 가상 클라이언트 · A/B/C/D →
    A 푸르트 · B 슬로우템포 · C 손결 · D 링크드마인드 중 본인이 고른 가상 클라이언트 1개가 정해져 있어야 합니다.
  • 04 🏷️ 17강 태그 5종 · 살아 있음 →
    high-intent·need-nurturing·vip-candidate·local-seoul·budget-low 다섯 태그가 태그 목록에 남아 있어야 오늘 분기가 작동합니다.

    Claude Code v2.1.46(2026-02-17 릴리스) 이상 설치 필요. 구버전이면 업데이트 후 진행합니다.

    OAuth(Open Authorization · 로그인 권한 위임 표준) 흐름으로 Claude에게 내 Google 캘린더 조회·생성 권한을 위임합니다.

📝 강사 노트
💬 그대로 읽으세요
톤: 이모지 최소, 공공 강의 담백체.

1. 강사가 전면 화면에서 6단계 중 핵심 3지점을 시연합니다(트리거 선택 → If/Else 분기 → Interactive Messenger 액션). 2. 시연 후 수강생은 각자 자리에서 본인 가상 클라이언트용 DM 자동 회신 워크플로우를 구축합니다. 3. Draft 단계에서 "Test Message" 기능으로 1회 테스트 발송. 실제 계정이 아닌 GHL 샌드박스에서 메시지 흐름 확인. 4. 완성된 수강생은 슬랙 `#수업실습` 채널에 `[19강][A/B/C/D] DM자동회신 · 워크플로우명` 형식으로 스크린샷 업로드. 진행 중 화면을 자유롭게 공유해 주시면 서로 참고가 됩니다. 5. 막히는 분은 `#강사-QnA` 채널 또는 손 들기로 강사·운영진 호출.

실습 A · DM 자동 회신

  • 01 💬 트리거 · 메시지 수신 →
    Instagram Message Received 또는 Facebook Message Received. 연동 안 되어 있으면 Customer Replied(웹챗) 대체.

    버튼 · 퀵리플라이 포함된 구조화 메시지 액션. 버튼 라벨 2~3개로 경로 나눕니다.
  • 03 🧭 6단계 · 실습 A 순서 →
    클라이언트 재확인 · 워크플로우 생성 · 트리거 · 조건 분기 · 액션 · Draft 테스트 순서로 Part 05에서 진행합니다.
