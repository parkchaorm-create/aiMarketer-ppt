  • 01 🎯 트리거 · 언제 · 오늘 4종 →
    Form Submitted · Contact Tag Added · Customer Booked Appointment · Instagram/Facebook Message Received 네 가지가 오늘 주로 쓰는 트리거입니다.
  • 02 🔀 조건 · 어떤 경우 · If/Else →
    태그 high-intent 보유 여부 · need-nurturing 보유 여부 · 영업 시간 외 같은 기준으로 경로가 갈라집니다.
  • 03 ⚙️ 액션 · 무엇을 · 핵심 4종 →
    Send Message · Interactive Messenger(버튼·퀵리플라이 포함) · Book Appointment · Add/Remove Contact Tag 네 가지가 오늘의 핵심 액션입니다.
CONCEPT · 01
📝 강사 노트
💬 그대로 읽으세요
액션(Action · 실제 동작) 은 흐름이 최종적으로 수행하는 일입니다. 19강에서 다루는 핵심 액션 네 가지:

1. Send Message (Instagram/Facebook/SMS/Email) — 메시지 자동 발송 2. Instagram Interactive Messenger · Facebook Interactive Messenger — 버튼·퀵리플라이가 포함된 구조화된 메시지 발송[^5] 3. Book Appointment — 지정 캘린더에 예약 자동 생성[^6] 4. Add/Remove Contact Tag — 17강에서 만든 태그 세트를 갱신

GHL은 네 가지 유형의 캘린더를 제공합니다[^7]. 1인 대행사 창업 초기에는 Simple Calendar 하나면 충분해요.

04 / 07

GHL 캘린더 · 오늘 쓸 것은 Simple

  • 01 ✅ Simple · 1인 · 1종 서비스 →
    1명 담당자 · 1종 서비스 예약에 최적화된 가장 기본 형태. 오늘 실습 B의 주 경로입니다.
  • 02 👥 Round Robin · 콜센터 방식 →
    여러 담당자에게 자동 배분되는 방식이에요. 콜센터가 돌아가며 응대하는 느낌. 오늘은 개념만 소개합니다.
  • 03 🎓 Class · 단체 수업 →
    한 시간대에 여러 명이 동시에 참여하는 단체 클래스·세미나에 맞춰진 캘린더. 오늘은 개념만 소개합니다.
  • 04 💇 Service · 조합 선택 →
    여러 서비스 + 담당자 조합 선택이 필요한 미용실·병원 스타일. 오늘은 개념만 소개합니다.
CONCEPT · 02
📝 강사 노트
💬 그대로 읽으세요
오늘은 Simple Calendar 한 개만 설정합니다. 나머지 세 종류는 "이런 것도 있다"는 정도로 알고 계시면 돼요.

잠깐 개념만 소개하고 지나갈게요. Conversation AI Autopilot은 설정한 지연 시간 이후에 AI가 자동으로 회신을 생성·발송하는 모드입니다[^3]. 영업 외 시간이나 집중 시간에 특히 유용해요. 다만 다음 이유로 오늘의 주 경로는 아닙니다.

학습 데이터(회사 정보·FAQ·톤) 사전 입력이 상세할수록 품질이 좋아짐

Part 03 · 세 원칙 정리

  • 01 🧪 가상 데이터 · 실제 금지 →
    테스트 발송 시 이메일은 test-<클라이언트>-01@example.com · 전화는 강사가 안내하는 GHL 샌드박스 번호만 사용합니다.
  • 02 📝 Save as Draft · 먼저 →
    모든 워크플로우는 Draft(초안 · 비공개) 상태로 시작. 테스트 완료 후에만 Publish합니다.
  • 03 🔗 취소 안내 · 한 줄 필수 →
    예약 확정 메시지에 "문의·취소는 이 링크로" 한 줄 포함. 고객 권리 안내는 자동 회신의 기본 예의입니다.

실습 시작 전 공통 주의 사항입니다. 세 가지만 지켜 주세요.
