실습 B · 태그 5개 배선

  • 01 🚦 Trigger · Filter · Action →
    워크플로우(Workflow · 자동화 흐름)의 3단 구조. 트리거(Trigger · 조건)·필터·액션(Action · 동작)을 조합해 태그를 자동화합니다.
  • 02 🏷️ 태그 5종 · 기본 세트 →
    vip-candidate · high-intent · need-nurturing · local-seoul · budget-low. 대행사 현장에서 가장 자주 쓰이는 다섯 묶음이에요.
  • 03 🧪 테스트 제출 · 옆자리 교환 →
    Publish 후 옆자리 수강생과 서로의 폼 URL로 가상 이메일을 제출하고, Contacts 탭에서 태그 부착을 확인합니다.

02 / 07

제출되는 순간 · 뭘 해 줄까요?

  • 01 🎬 제출 시점 · 규칙 발동 →
    앞서 만든 폼이 제출되는 순간부터 규칙에 따라 태그가 자동으로 붙도록 설정하는 게 이번 파트의 과제입니다.
  • 02 🗺️ 워크플로우 · 조건·동작 흐름도 →
    GHL에서는 "이런 일이 생기면 → 저런 일을 해 달라"를 워크플로우(Workflow) 메뉴에서 그려요. 조건·동작 흐름도라고 생각하시면 됩니다.
  • 03 ✍️ 오늘 흐름 · 한 줄 요약 →
    트리거 "폼 제출됨" → 필터(응답 확인) → 액션(조건에 맞는 태그 추가). 이 한 줄을 5개 규칙으로 복제합니다.

03 / 07

규칙 1~3 · VIP·의향·육성

  • 01 ⭐ vip-candidate · 예산 상위 →
    예산 필드 응답 = "월 100만 원 이상" 또는 상위 구간 → vip-candidate 태그. 담당자 우선 응대 큐에 진입합니다.
  • 02 ⚡ high-intent · 즉시 시작 →
    시작 시기 = "즉시 시작" 또는 "이번 주 내" → high-intent 태그. 빠른 응답·견적서 템플릿 발송 준비 대상.
  • 03 🌱 need-nurturing · 검토 중 →
    시작 시기 = "정보 수집 중"·"검토 단계" → need-nurturing. 뉴스레터·콘텐츠 드립(Drip · 정기 발송) 대상입니다.

04 / 07

규칙 4·5 · 지역·예산

  • 01 📍 local-seoul · 특정 지역 →
    지역 필드 = "서울" (또는 본인 가상 클라이언트 주 서비스 지역) → local-seoul 태그. 지역 한정 프로모션·오프라인 이벤트 대상.
  • 02 💰 budget-low · 예산 제한 →
    예산 = "10만 원 이하" 또는 "미정" → budget-low 태그. 소형 상품·체험 패키지 안내 대상이며 즉시 영업은 제외합니다.
  • 03 ⚠️ 중요 · 배제 아닌 분류 →
    budget-low는 배제 태그가 아닙니다. "지금 단계에 맞는 서비스를 제안하기 위한 분류". 태그의 용도는 배제가 아니라 분류예요.

05 / 07

Publish 후 · 무엇을 볼까요

  • 01 ✅ 성공 신호 · Contacts Tags →
    테스트 제출 후 Contacts 탭에 컨택트가 생성되고, 상세 화면 우측 Tags 영역에 의도한 태그가 자동으로 들어 있으면 성공.
  • 02 ❌ 실패 · 3가지 순서로 점검 →
    ①워크플로우가 Published 상태인지 ②트리거에 올바른 폼이 연결됐는지 ③If 조건 필드명이 폼 필드와 정확 일치하는지.
  • 03 🧷 주의 · 가상 이메일만 →
    테스트 제출 시 test-pureut-01@example.com 같은 가상 이메일 사용. 실제 개인정보·본인 이메일은 금지입니다.
