실습 A · 랜딩 폼 1개 만들기

  • 01 🧭 6단계 지도 · 큰 그림 →
    GHL(GoHighLevel) 폼 빌더(Form Builder)에서 클라이언트 확인부터 공개 URL 발급까지 6단계로 진행합니다.
  • 02 🤖 클로드 코드 · 필드 초안 →
    클로드 코드에게 필드 초안 프롬프트(Prompt · AI에게 주는 지시문)를 주고, GHL UI로 옮길 재료를 받습니다.
  • 03 📤 공개 URL · 슬랙 업로드 →
    발급된 공개 URL을 슬랙 #수업실습 채널에 [17강][A/B/C/D] 형식으로 업로드해 공유합니다.

02 / 07

필드 어색해도 · 괜찮을까요?

  • 01 🚪 입구부터 · 만들어야 →
    "리드가 스스로 라벨 달기"의 출발점은 정보를 받아줄 입구입니다. 그 입구가 바로 폼(Form · 입력 양식)이에요.
  • 02 🧰 GHL · 네이티브 폼 빌더 →
    GHL은 이름·이메일부터 드롭다운·다중 선택까지 네이티브 폼 빌더를 내장하고 있어요. 조건부 로직과 커스텀 CSS도 가능합니다.
  • 03 ⛑️ 실패 안전망 · 몇 번이든 →
    첫 시도에서 필드 순서가 어색해도 괜찮습니다. 공개 전까지 몇 번이든 수정할 수 있어요. 작동 여부만 우선 확인합니다.

CONCEPT · 6단계 전개
03 / 07

1~3단계 · 클라이언트·초안·생성

  • 01 👤 1단계 · 가상 클라이언트 →
    1강에서 선택한 A/B/C/D 중 하나를 확인합니다. 폼 문구는 선택한 브랜드 톤에 맞춰 담백하게 구성합니다.
  • 02 🤖 2단계 · 클로드 코드 초안 →
    "푸르트 꽃 구독 리드 폼 · 필드 6개 · 한국어 · 담백" 같은 프롬프트로 필드 목록 초안을 받습니다. GHL UI에 옮길 재료용이에요.
  • 03 🆕 3단계 · Sites → Forms → New →
    GHL 좌측 메뉴 Sites → Forms → New Form. Blank 또는 Template 선택 후 필드를 드래그&드롭으로 배치합니다.

04 / 07

4~6단계 · 설정·메시지·URL

  • 01 ⚙️ 4단계 · 필드 타입·필수 →
    이메일·이름은 필수, 관심 분야는 드롭다운, 예산·의향은 라디오. 필드명은 Part 05 태그 규칙과 동일 키워드로 지어야 연결이 쉬워요.
  • 02 💬 5단계 · 감사 메시지 →
    "접수되었습니다. 영업일 기준 2~3일 내 연락드리겠습니다." 같은 담백한 문구. 리다이렉트(Redirect · 이동) URL은 선택.
  • 03 📤 6단계 · Share · 공개 URL →
    Share 버튼 → 공개 URL 복사. 슬랙 #수업실습 채널에 [17강][A/B/C/D] 폼URL · 한 줄 설명 형식으로 업로드합니다.

05 / 07

막히는 4가지 · 1차 조치

  • 01 🔍 New Form 버튼 · 안 보여요 →
    좌측 메뉴 Sites 진입이 안 된 상태입니다. Sites → Forms → All Forms 순서를 확인하고 다시 클릭하세요.
  • 02 💾 저장이 · 안 돼요 →
    우측 상단 Save 버튼 누락이 가장 흔한 원인. Save 클릭 후 미리보기 재시도하면 대부분 해결됩니다.
  • 03 🚫 공개 URL · 404 →
    Publish(게시) 단계 누락이에요. Share → Publish를 먼저 실행한 뒤 URL을 다시 열어 확인하세요.
  • 04 ✉️ 이메일 · 이상한 값 →
    이메일 Validation(검증) 미체크. 필드 설정에서 Email 타입 + 필수 체크를 켜면 잘못된 형식은 자동 거부됩니다.
